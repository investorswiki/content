# Bloquear
En resumen, el término bloque se refiere a archivos de computadora que almacenan datos de transacciones. Estos bloques están dispuestos en una secuencia lineal que forma una cadena interminable de bloques, de ahí el término [blockchain](/blockchain).

Por lo tanto, toda la información sobre las transacciones de blockchain se recopila y registra dentro de estos bloques, y cada bloque recién generado se conecta al anterior mediante el uso de técnicas [criptográficas .](/cryptography)

La cadena de bloques vinculados almacena todos los datos de transacciones generados desde el lanzamiento de una cadena de bloques en particular. Entonces, los registros se remontan al primer bloque, que se conoce como bloque cero o [bloque génesis](/genesis-block). El número de bloques confirmados desde el bloque de génesis se indica como la altura del [bloque](/block-height) [k](/block-height).

Tomando como ejemplo la cadena de bloques de Bitcoin, los bloques se componen de múltiples elementos. Los bloques de [Bitcoin](/bitcoin) contienen (entre otras cosas) una lista de transacciones recientes, una marca de tiempo y una referencia al bloque anterior. Esta referencia es un hash criptográfico de los datos del bloque anterior.

La generación de un nuevo bloque siempre incluye el hash del anterior, y esto es lo que hace que los bloques estén criptográficamente enlazados. Tal estructura permite la creación de una base de datos segura que es altamente resistente a la manipulación y los ataques. El hash de bloque actúa como un identificador. Es único para cada bloque y se produce a través de un proceso llamado [minería](/mining).

Básicamente, el hash de bloque es la solución a un problema matemático complejo, y al minero que encuentra una solución válida para el siguiente bloque se le otorga el derecho de validar ese bloque (y las transacciones en él). Dado que la minería requiere una cantidad significativa de recursos computacionales, los bloques extraídos con éxito producen nuevos Bitcoins como una forma de recompensar a los mineros por su trabajo (ver [recompensa por bloque](/block-reward) ).

Pero a veces, dos o más mineros pueden encontrar un hash de bloque válido al mismo tiempo, por lo que se transmiten dos bloques diferentes a la red. Esto lleva a la creación de dos cadenas competidoras. Para abordar este problema, los participantes de la red ( [nodos](/node) ) elegirán la cadena que eventualmente se convierta en la más larga (con más “trabajo acumulado”). Luego, la otra cadena se descarta y sus bloques se convierten en bloques obsoletos (ver [bloques huérfanos](/orphan-block-cryptocurrency) ).

Aunque los bloques generalmente se analizan en el contexto de las transacciones de criptomonedas, también pueden estar relacionados con otros tipos de datos digitales que se almacenan en un sistema de cadena de bloques.

## Reflejos

- Un bloque se refiere a un comercio de gran volumen que se produce a la vez.

- Las transacciones en bloque a veces se realizan fuera de los mercados abiertos para disminuir el impacto en el precio del valor.

- Para no influir en los precios del mercado, las órdenes de bloques grandes pueden dividirse en órdenes más pequeñas y ejecutarse a través de diferentes corredores para enmascarar el tamaño real.

- Los intercambios suelen definir un bloque como más de 10 000 acciones o una operación que tiene un valor nominal superior a $200 000.
