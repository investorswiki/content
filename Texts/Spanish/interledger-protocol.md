# Protocolo Interledger
## ¿Qué es el protocolo Interledger?

El protocolo Interledger es un protocolo de cadena de [bloques](/blockchain) que se utiliza para pagos en diferentes redes de pago. El protocolo [de código abierto](/open-source) conecta los libros de contabilidad de dos o más bancos diferentes, eliminando así intermediarios y autoridades centrales del sistema. Promete reducir los costos y el tiempo requerido para procesar transacciones transfronterizas.

Interledger es utilizado por Ripple Labs para conectar sistemas bancarios a través de las fronteras donde el token [Ripple](/ripple-cryptocurrency) (XRP) funciona como una capa de liquidación estandarizada entre bancos globales, lo que hace que Ripple se parezca un poco a un servicio digital [hawala](/hawala).

## Entendiendo el Protocolo Interledger

El proceso actual de transferencias transfronterizas entre bancos implica la navegación de múltiples protocolos de pago utilizados por los libros de contabilidad digitales para procesar transacciones. Los protocolos se comunican mediante conectores, pero las herramientas y los estándares utilizados en dichos sistemas están fragmentados.

Por ejemplo, el conjunto actual de conectores no tiene modos estandarizados para la comunicación. La introducción de bancos intermediarios para facilitar las transacciones entre entidades que no tienen una relación directa entre sí complica aún más el proceso. Esto se debe a que multiplica la cantidad de saltos para completar una transacción, lo que la hace más costosa y lenta. También magnifica los riesgos de seguridad porque los pagos tienen que viajar a través de múltiples sistemas para llegar al destinatario final.

El Protocolo Interledger utiliza el concepto de [custodia criptográfica](/escrow) para permitir el paso de fondos a través de conectores o nodos en la red. El documento técnico del protocolo Interledger define la custodia criptográfica como el equivalente financiero de un protocolo de compromiso de dos fases. El último protocolo consta de dos pasos:

1. El primer paso consiste en definir un conjunto de condiciones para que una transacción avance o se cancele.

1. El segundo paso define el procesamiento de la transacción una vez que se cumplen las condiciones.

El depósito en garantía criptográfico para libros mayores es el bloqueo condicional de fondos entre dos partes. Los fondos se liberan solo después de que se cumplen ciertas condiciones, incluidas las basadas en el tiempo para que se lleve a cabo la transacción. De lo contrario, la transacción se anula. Al final de una transacción de pago exitosa, el remitente recibe un recibo criptográfico del destinatario. Alternativamente, los fondos depositados se devuelven a las partes respectivas.

## Modo atómico frente a modo universal

El protocolo Interledger se puede implementar en dos modos: modo atómico y modo universal. En modo atómico, los notarios se incorporan al sistema. Son un grupo ad-hoc que se utiliza para verificar y validar transacciones. Por lo general, los modos atómicos tienen lugar entre nodos de conexión confiables entre bancos o compañías de servicios financieros que pueden relacionarse entre sí .

El modo universal no requiere notarios y puede funcionar entre conectores que no son de confianza. Utiliza la criptomoneda interna de Ripple, XRP, para facilitar las transferencias. La transferencia va acompañada de limitaciones de tiempo. Si no se lleva a cabo dentro de un plazo determinado, la transacción se anula .

## Reflejos

- Interledger es un conjunto de protocolos de código abierto para enviar pagos a través de diferentes libros de contabilidad.

- Basado en la tecnología blockchain, el objetivo es eliminar intermediarios para crear una red de pagos interbancarios segura, descentralizada y transfronteriza.

- Interledger no está vinculado a una sola empresa, cadena de bloques o moneda, aunque se destaca de manera destacada con la criptomoneda Ripple.
