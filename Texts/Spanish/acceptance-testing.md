# Test de aceptación
## ¿Qué son las pruebas de aceptación?

La prueba de aceptación, en el contexto de las industrias de ingeniería y software, es una prueba funcional que se realiza en un producto o prototipo antes de comercializarlo o entregarlo, para decidir si se han cumplido las especificaciones o el contrato. También se asegura de que la calidad y el diseño del producto cumplan con las obligaciones contractuales y reglamentarias en términos de funcionalidad, facilidad de uso, durabilidad y seguridad.

Si se determina que un producto es inaceptable en esta etapa, puede devolverse para su modificación, depuración, reparación o rediseño antes de que se convierta en una tarea costosa para el productor, como sería el caso de una retirada del producto.

## Comprender las pruebas de aceptación

El proceso de prueba de aceptación, que también se conoce como prueba [de usuario final](/end-user),. prueba de aceptación operativa o prueba de campo, actúa como una forma de control de calidad inicial para identificar problemas y defectos mientras aún se pueden corregir de forma relativamente sencilla y económica.

A menudo implica probar un producto prototipo o uno que aún no se ha producido en masa para el mercado de consumo. También es una de las etapas finales del ciclo de prueba de una parte del software y, a menudo, ocurre antes de que un cliente acepte la nueva aplicación, y fomenta una estrecha colaboración entre los desarrolladores y los clientes.

Las pruebas de aceptación están diseñadas para replicar el uso real previsto del producto para verificar que es completamente funcional y cumple con las especificaciones acordadas entre el cliente y el fabricante. Estos pueden incluir pruebas químicas, pruebas físicas o pruebas de rendimiento, que pueden refinarse e iterarse si es necesario. Si los resultados reales coinciden con los resultados esperados para cada caso de prueba, el producto pasará y se considerará adecuado. Entonces será rechazado o aceptado por el cliente. Si se rechaza, puede repararse o abandonarse por completo si las correcciones requeridas resultaran demasiado costosas o llevaran mucho tiempo.

## Ejemplos de pruebas de aceptación

Los tipos de pruebas de aceptación incluyen:

- Pruebas alfa y beta

- Pruebas de Aceptación de Contratos

- Pruebas de Aceptación de Reglamentos

- Pruebas de Aceptación Operacional

Las pruebas alfa y beta son ejemplos de pruebas de aceptación. Las pruebas alfa son internas y tienen como objetivo detectar cualquier defecto evidente, mientras que las pruebas beta son una prueba piloto externa de un producto antes de que entre en producción comercial.

Las pruebas de contrato garantizan que los proveedores, vendedores o fabricantes que se han registrado como contratistas del proceso de producción han cumplido las especificaciones de un producto. Si algo no cumple con las obligaciones establecidas en el contrato, se debe rectificar o se pueden emprender acciones legales.

De manera similar, el gobierno o algún otro organismo regulador puede exigir ciertas características de seguridad o controles de calidad que deben cumplirse antes de que pueda venderse al público. El incumplimiento de las pautas reglamentarias puede resultar en multas, retiros del mercado u otras acciones legales.

## Reflejos

- Al probar un producto o pieza de software antes de que se produzca en masa, se pueden realizar correcciones de errores u otras modificaciones de una manera más rentable.

- Existen varios métodos para las pruebas de aceptación, incluidas las pruebas alfa/beta, que implementan un producto en una fase alfa inicial para detectar errores obvios, seguido de su presentación a los probadores beta para identificar errores menores o más matizados.

- Las pruebas de aceptación evalúan un producto, prototipo o aplicación de software para asegurarse de que cumple con las especificaciones y la calidad antes de ponerlo en producción.
