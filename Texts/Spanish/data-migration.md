# Migración de datos
## ¿Qué es la migración de datos?

La migración de datos es el proceso de mover información digital almacenada entre computadoras, sistemas o formatos. La migración de datos ocurre por varias razones, incluido el reemplazo o el mantenimiento del servidor, un cambio de [centros de](/data-warehousing) datos, proyectos de consolidación de datos y actualizaciones del sistema. Dado que gran parte del conocimiento corporativo y la inteligencia comercial de una empresa [están](/business-intelligence-bi) [contenidos](/business-intelligence-bi) en sus datos, cualquier proyecto de migración de datos debe realizarse con cuidado para minimizar los riesgos.

## Comprender la migración de datos

La migración de datos representa un riesgo importante para [la continuidad del negocio](/business-continuity-planning) si no se realiza correctamente. La pérdida de datos es, por supuesto, el peor de los casos, pero las empresas también deben lidiar con el tiempo de inactividad, los problemas de compatibilidad y los problemas generales de rendimiento del sistema. La migración de datos se hace más difícil debido a las grandes cantidades de datos, los diversos formatos de datos y los hábitos de datos divergentes dentro de una corporación.

Para minimizar los riesgos de la migración de datos, las empresas crean políticas de migración de datos detalladas que priorizan las [copias de seguridad](/back-up),. el orden de movimiento y los entornos de datos simultáneos cuando sea posible. Si una empresa no puede ejecutar un entorno previo a la migración mientras se prepara el nuevo entorno, habrá un tiempo de inactividad significativo a medida que se suspendan las operaciones comerciales en las aplicaciones actuales para permitir la migración de datos. Este tipo de migración de datos de parada, transferencia e inicio puede ser necesario cuando se cambia a nuevas plataformas o cuando hay límites estrictos en el almacenamiento físico y se necesitan intercambios o correcciones en la tecnología de almacenamiento existente.

## Migración de datos sin tiempo de inactividad

El modelo de migración sin tiempo de inactividad depende de tener suficiente almacenamiento para crear y ejecutar dos entornos completos. Una copia completa de los datos de una empresa se lleva al nuevo entorno y se prueba mientras los empleados permanecen en el antiguo entorno. Los errores se resuelven en el nuevo sistema, lo que garantiza que todas las aplicaciones sigan funcionando y que todo esté donde debería estar. Una vez completada la prueba, se trae una copia nueva y todos los empleados se cambian al nuevo entorno. El entorno de datos anterior a veces se deja abierto durante un período de meses para que los empleados puedan obtener archivos del sistema de datos anterior pero no escribir datos nuevos en esos servidores. En todas las migraciones de datos, se realiza una [auditoría de datos posterior a la migración](/audittrail) para verificar la pérdida de datos.

## Mejora de la migración de datos

Una cosa que puede mejorar una migración de datos es limpiar y estandarizar las prácticas de datos antes de la migración. La organización de los datos de una empresa suele ser un reflejo de los diferentes hábitos de archivo de su gente. Dos personas con el mismo rol bien pueden usar prácticas completamente diferentes. Por ejemplo, guardar contratos por proveedor en un caso y por [año fiscal](/fiscalyear) y mes en otro. La unificación de las prácticas de datos puede ser un trabajo mucho más grande que la migración real de los datos, pero los datos limpios y coherentemente organizados respaldados por políticas claras ayudan a preparar los datos de una empresa para el futuro para muchas migraciones por venir.

## Reflejos

- La migración de datos se refiere a la transición de la información almacenada en un sistema informático a otro.

- Este proceso puede ocurrir debido a actualizaciones de tecnología, movimiento entre ubicaciones físicas o durante operaciones de mantenimiento de rutina.

- Debido a que las empresas de hoy en día dependen en gran medida de sus datos para llevar a cabo sus actividades comerciales diarias, la migración de datos debe llevarse a cabo de forma cuidadosa y segura para no causar interrupciones, pérdida de datos o brechas de seguridad.
