# Progressiv nettapplikasjon (PWA)
Begrepet Progressive Web Application (PWA) refererer til en spesiell type applikasjon som er laget ved bruk av moderne nettteknologi og som følger grunnleggende nettstandarder. I hovedsak er en PWA en nettapp som fungerer omtrent som en innebygd app. Men hva er forskjellene mellom web- og native-applikasjoner?

### Nettapp vs. opprinnelig app

På den ene siden er nettapper mer tilgjengelige (spesielt for nye brukere) siden de lett kan nås via en nettleser. Når du bruker nettapper, trenger ikke brukere å installere applikasjonen på datamaskinen eller mobilenheten. Web-apper er også vert online, på en webserver, noe som betyr at de enkelt kan oppdateres og brukes i alle slags enheter og operativsystemer. Med andre ord er det ikke nødvendig å sende oppdateringer til flere appbutikker eller til brukere direkte.

Innfødte apper, derimot, er de som brukere må laste ned og installere lokalt på enhetene sine. Dette er appene vi jevnlig laster ned via appbutikker eller appmarkedsplasser. Generelt har native apper høyere hastighet og ytelse enn nettapper fordi de kan fungere bedre synkront med enhetens maskinvare og native funksjoner (inkludert kamera, GPS og mikrofon). Innfødte apper er imidlertid dyrere å utvikle og er ikke alltid kompatible med alle enheter (hvert operativsystem trenger sin egen versjon av appen).

### Progressive Web Application (PWA)

Kort sagt, progressive nettapplikasjoner (PWA) er pålitelige, raske og engasjerende. De samler det beste fra både native apper og nettapper. Så til tross for at det er en nettapplikasjon, er PWA-er kompatible med alle enheter og ser i praksis mer ut som en innebygd app enn en nettleserapplikasjon. PWA-er kan fungere på alle nettlesere som overholder de mest grunnleggende nettleserstandardene.

Vanligvis bruker PWA-er teknologier som HTML, CSS og JavaScript, som lar dem operere med økt hastighet og med mye flere funksjoner. Som navnet antyder, kan PWAer forbedres gradvis, for hele levetiden.

I motsetning til konvensjonelle nettapper, er ikke PWA-er strengt avhengig av en internettforbindelse, noe som betyr at de kan fortsette å kjøre under dårlige nettverksforhold eller til og med offline. Dessuten kan PWA-er kjøre i bakgrunnen, er alltid oppdaterte og kan enkelt deles med en enkel lenke.

Bortsett fra det teller PWA-er mot nettrafikk, noe som betyr at de, i motsetning til native apper, kan øke analysene til en nettside når det gjelder sidevisninger, noe som er gunstig for deres generelle annonser og søkeytelse.
