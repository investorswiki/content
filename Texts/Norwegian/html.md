# HyperText Markup Language – HTML
## Hva er HyperText Markup Language (HTML)?

HyperText Markup Language (HTML) er settet med markup-symboler eller -koder som er satt inn i en fil beregnet for visning på Internett. Markeringen forteller nettlesere hvordan de skal vise en nettsides ord og bilder.

Hver enkelt brikkekode (som vil falle mellom "<" og ">"-tegn) blir referert til som et element, selv om mange mennesker også refererer til det som en tag. Noen elementer kommer i par som indikerer når en visningseffekt skal begynne og når den skal slutte.

## HTML forklart

HyperText Markup Language er datamaskinspråket som gjør det lettere å lage nettsider. Språket, som har kodeord og syntaks akkurat som alle andre språk, er relativt enkelt å forstå og, etter hvert som tiden går, stadig sterkere i det det lar noen lage. HTML fortsetter å utvikle seg for å møte kravene og kravene til Internett under dekke av World Wide Web Consortium, organisasjonen som designer og vedlikeholder språket; for eksempel med overgangen til [Web 2.0](/web-20).

HyperText er metoden som Internett-brukere navigerer på nettet på. Ved å klikke på spesialtekst kalt hyperlenker, bringes brukere til nye sider. Bruken av hyper betyr at den ikke er lineær, så brukere kan gå hvor som helst på Internett ved å klikke på de tilgjengelige koblingene. Markup er hva HTML-tagger gjør med teksten inne i dem; de merker det som en bestemt type tekst. Markeringstekst kan for eksempel komme i form av fet skrift eller kursiv skrift for å trekke spesiell oppmerksomhet til et ord eller en setning.

## Grunnleggende om HyperText Markup Language

I kjernen er HTML en serie korte koder som skrives inn i en tekstfil. Dette er kodene som driver HTMLs muligheter. Teksten lagres som en HTML-fil og vises i en nettleser. Nettleseren leser filen og oversetter teksten til en synlig form, som anvist av kodene forfatteren brukte for å skrive det som blir den synlige gjengivelsen. Å skrive HTML krever at tagger brukes riktig for å skape forfatterens visjon.

Taggene er det som skiller normal tekst fra HTML-kode. Tagger er ordene mellom det som kalles vinkelparenteser, som lar grafikk, bilder og tabeller vises på nettsiden. Ulike tagger utfører forskjellige funksjoner. De mest grunnleggende kodene bruker formatering på tekst. Ettersom nettgrensesnitt må bli mer dynamiske, kan Cascading Style Sheets (CSS) og JavaScript-applikasjoner brukes. CSS gjør nettsider mer tilgjengelige og JavaScript gir kraft til grunnleggende HTML.

## HTML vs. XML

I motsetning til HTML, lar Extensible Markup Language eller XML brukere definere sin egen markup. Ved å bruke XML kan for eksempel én bruker velge å angi en fotnote med taggen <footnote>, mens en annen bruker kan velge <fn>.

Ved å bruke HTML kan bare én forhåndsbestemt kode brukes for å angi en bestemt type informasjon. XML-dokumenter er ment å være enkle å lese siden de inneholder brukerdefinerte tagger og siden dokumentene kun består av markering og innhold.

## Høydepunkter

- HyperText Markup Language (HTML) er det grunnleggende skriptspråket som brukes av nettlesere for å gjengi sider på verdensveven.

- Tidlige versjoner av HTML var statiske (Web 1.0), mens nyere iterasjoner har mye dynamisk fleksibilitet (Web 2.0, 3.0).

- Markup er teksten som vises mellom to spisse parenteser (f.eks. <fotnote>), og innhold er alt annet.

- HyperText lar en bruker klikke på en lenke og bli omdirigert til en ny side som den lenken refererer til.
