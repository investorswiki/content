# 付加価値通信網（VAN）
##付加価値通信網（VAN）とは何ですか？

付加価値通信網（VAN）は、企業がデータを送信して[相手方と安全に共有する方法を提供する、プライベートなホスト型サービスです](/counterparty)。付加価値通信網は、企業間の電子データ交換（EDI）を容易にする一般的な方法でした。インターネットが安全な電子メールの出現でこのサービスの競争を生み出したので、VANは、メッセージの暗号化、安全な電子メール、管理レポートなどを含むようにサービスの提供を拡大することで対応しました。

付加価値通信網は、企業が通信する必要のある関係者の数を減らすことにより、通信プロセスを簡素化します。 VANは、標準ベースまたは独自のデータを共有するビジネスパートナー間の仲介役として機能することでこれを実現します。 VANには監査機能が設定されているため、交換されるデータは正しくフォーマットされ、次のパーティに転送される前に検証されます。 VANは、付加価値ネットワークまたはターンキー通信回線と呼ばれることもあります。

##付加価値通信網（VAN）の仕組み

付加価値通信網は、一般に、サプライヤとの効率的な[サプライチェーン管理のために大企業](/scm)によって、または業界コンソーシアムや電気通信会社によって使用されます。 VANは通常、メールボックス設定で動作します。この設定では、会社がトランザクションをVANに送信し、VANがそれを受信者のメールボックスに配置します。受信者はVANに接続してトランザクションを取得し、独自のトランザクションを送信します。

このシステムは、非構造化テキストではなく標準化された構造化データに使用されることを除けば、電子メールに似ています。

##インターネット時代のVAN

インターネットの普及により、主にコストを考慮して、VANの魅力が低下しています。簡単に言えば、通常のVAN契約に含まれる最低月額料金と文字ごとの料金を支払うよりも、インターネットを介してデータを移動する方が費用効果が高いことがよくあります。 VANは、ヘルスケア、小売、製造などの特定の業界に焦点を当てることで、インターネットからの課題に対抗してきました。これらの業界には、VANを真の付加価値ソリューションにする独自のデータ整合性とセキュリティ上の懸念があります。

> VANは、会社がより少ない関係者と通信できるようにすることで、通信プロセスを簡素化します。

>>

、たとえば[エンタープライズリソースプランニング（ERP）スイート](/erp)に直接送信されるようにフォーマットできます。この2つの会社間の直接交換により、商取引の速度が向上すると同時に、手動のデータ入力で発生する人的エラーの可能性が減少します。

VANは、データの配信ステータスと対応するワークフローを表示する可視化ツールも提供できるため、企業は電話や電子メールを交換するのではなく、システムを介して依存するアクティビティをより適切に調整できます。 VANをより効率的かつ正確に使用するだけでなく、情報交換のために人間のデータ入力の専門家を雇うコストを節約できます。

多くのインターネット以前のテクノロジーと同様に、VANは、今後も関連性を維持するために、自らを改革しなければなりませんでした。現在、VANは、 [EDIの交換](/interchange)と取得、メッセージの認証、および過去のトランザクションのアーカイブのために、メールボックスを超えたサービスを提供しています。最新のVANは、EDIデータの自動バックアップ、安全なWebポータルを介したそのデータへの柔軟なアクセス、および無制限のデータ価格設定パッケージを提供することにより、ビジネスに価値をもたらします。

##ハイライト

-VANは、サプライチェーンを管理するために重要です。

-付加価値通信網は、企業間での電子データ交換によく使用されます。

-VANは、少数の関係者間の通信により、通信プロセスを容易にします。
