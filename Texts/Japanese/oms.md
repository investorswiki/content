# 注文管理システム（OMS）
##注文管理システム（OMS）とは何ですか？

、効率的かつ費用効果の高い方法で[証券注文](/security)を実行するために開発された電子システムです。ブローカーやディーラーは、さまざまな種類の証券の注文を処理するときにOMSを使用し、システム全体で各注文の進行状況を追跡できます。

金融市場のOMSは、取引注文管理システムと呼ばれることもあります。

企業（特に[eコマースビジネスと販売者）もOMSを使用して、販売](/ecommerce)[時点から](/point-of-sale)顧客への配送までの販売と履行のプロセスを合理化および自動化します。

##注文管理システム（OMS）を理解する

OMSは、取引注文の実行を容易にし、管理するソフトウェアシステムです。金融市場では、証券の買い注文または売り注文を実行するために、取引システムで注文を行う必要があります。

[取引注文](/order)には通常、次の情報が含まれています。

-セキュリティ識別子（つまり、その[ティッカー](/stocksymbol)シンボル）

-注文タイプ（つまり、購入、販売、またはショート）

-注文サイズ

-注文タイプ（例：[市場](/marketorder)、[制限](/limitorder)、[停止](/stoporder)など）

-注文手順（例：[日注文](/dayorder)、[記入または殺害](/fok)、[キャンセル済み](/gtc)など）

-注文送信（ブローカー、[電子通信ネットワーク）](/ecn) [ECN]、[at-the-close](/at-the-close-order) [ATC]など）

[Financial Information eXchange](/financial-information-exchange) （FIX）プロトコルを使用してソフトウェアシステムを介して取引を実行します。 FIXは、数兆ドルの証券取引と市場に関連する国際的なリアルタイムの交換情報を共有するために使用される電子通信プロトコルです。

ただし、トランザクションの通信は、カスタム[アプリケーションプログラミングインターフェイス](/application-programming-interface)（API）を使用して実行することもできます。 FIXプロトコルは、OMSを使用して、[ヘッジファンド](/hedgefund)と投資会社を世界中の何百ものカウンターパーティにリンクします。

##特別な考慮事項

機関投資家のトレーディングデスクでは、OMSをバイサイドとセルサイドの両方で使用して、企業がトレードのライフサイクルを管理し、ポートフォリオ全体の投資を自動化および合理化できるようにすることができます。

レビューのために、[バイサイド](/buyside)は、投資信託、年金基金、および資金管理の目的で証券の大部分を購入する傾向がある保険会社などの投資機関で構成されるウォール街のセグメントです。

バイサイドは[セルサイドの反対](/sellside)です。セルサイドは直接投資を行いません。代わりに、[アップグレード](/upgrade)、[ダウングレード](/downgrade)、目標価格、およびその他の意見に関する投資の推奨事項を投資市場に提供します。

一緒に、バイサイドとセルサイドはウォール街の両側を構成します。

##証券取引OMS

OMSで取引または監視できる多くの商品や証券があります。 OMSを使用して取引される金融商品には次のものがあります。

-エクイティ

-債券などの債券商品

-通貨

-原油や銅などの商品

-ローン

- 現金

-金利と通貨のオプションで構成される可能性のあるデリバティブ

通常、エクスチェンジメンバーのみがエクスチェンジに直接接続できます。つまり、セルサイドOMSは通常エクスチェンジ接続を備えていますが、バイサイドOMSはセルサイド企業への接続に関係しています。セルサイドで注文が実行されると、セルサイドOMSはその状態を更新し、実行レポートを注文の元の会社に送信する必要があります。

>取引OMSは、多くの場合、価格と実行の観点から最良の取引所に注文をルーティングするか、トレーダーが注文を送信する取引所を[手動でルーティングできるようにします。](/directedorder)

>>

OMSはまた、企業がシステムに入力された注文に関する情報にアクセスできるようにする必要があります。これには、すべての[未処理の注文](/openorder)や以前に完了した注文の詳細が含まれます。 OMSは、意図された資産配分アクションをバイサイドの市場性のある注文に変換することにより、ポートフォリオ管理をサポートします。

##トレーディングOMSのメリット

多くのOMSは、リアルタイムの取引ソリューションを提供します。これにより、ユーザーは、リアルタイムの価格ストリーミングによって、市場価格を監視し、すべての市場にわたる複数の取引所で注文を瞬時に実行できます。企業がOMSから達成できる利点には、注文の管理やポートフォリオの資産配分などがあります。

効果的なOMSは、参入前と参入後の両方での取引のリアルタイムチェックを含む、規制順守を支援する上で重要です。 OMSは、コンプライアンス担当者が取引のライフサイクルを追跡して、違法行為や金融詐欺、および会社の従業員による規制違反がないかどうかを判断するのに役立ちます。 OMSは、ポートフォリオマネージャー、トレーダー、コンプライアンス担当者間のワークフローとコミュニケーションを改善できます。

OMSは、ポジションのリアルタイムモニタリング、規制違反を防止する機能、取引実行の速度と正確さ、および結果として生じる大幅なコスト削減のために、金融サービス業界における重要な発展です。

##ビジネスOMS

OMSの取引に加えて、注文管理には他にもいくつかのコンテキストがあります。企業はOMSを使用して、POSから配達までの顧客の注文を追跡し、返品と返金を処理できます。これは、販売量が多い企業やeコマースによる配送に依存している企業に特に役立ちます。

>オンラインストアを持っているか、AmazonやeBayなどのサイトでeコマースを行っている人にとって、OMSを導入すると、エラーを減らし、時間を節約し、収益性を高めるのに大いに役立ちます。これらや他のオンラインマーケットプレイスと簡単に統合できるターンキーOMSプラットフォームがいくつかあります。

>>

したがって、OMSの選択は、関係するビジネスの種類、規模、および範囲によって異なります。コストのかかるシステムには、さまざまな通貨での注文支払いの受け取り、場所や近接性に基づくサプライヤーと倉庫のルーティング、顧客の注文状況の追跡、在庫状況の予測による供給不足の可能性の予測、請求、返品/交換。

##ハイライト

-金融市場では、証券の買い注文または売り注文を実行するために、取引システムで注文を行う必要があります。

-注文管理システム（OMS）は、取引注文の実行を容易にし、管理するソフトウェアシステムです。

-効果的なOMSは、企業がポジションをリアルタイムで監視し、規制違反を防止するのに役立ちます。

-ブローカーやディーラーは、さまざまな種類の証券の注文を処理するときにOMSを使用し、システム全体で各注文の進行状況を追跡できます。

＃＃よくある質問

###なぜ企業はOMSを必要とするのですか？

企業は、注文処理プロセスを合理化することにより、OMSの恩恵を受けます。これにより、POSから配送まですべてを管理できます。 eコマースの売り手は、ロジスティクス、出荷、返品、Amazon、eBay、AliExpressなどのプラットフォームとのインターフェースを自動化できるOMSの恩恵を受けることができます。

###トレーダーがOMSを必要とするのはなぜですか？

OMSは、トレーダーが単純なものから複雑なものまで、より効率的に注文を入力して実行するのに役立ちます。これにより、トランザクションコストが削減され、[最高の](/bestexecution)[実行が可能](/bestexecution)になり、エラーが減少します。また、[フィル](/fill)を報告し、取引を予約し、ポジションまたはポートフォリオを更新します。一部のOMSは、取引戦略やストップロスや[トレーリング](/trailingstop)ストップなどのリスク軽減策を自動化することもできます。

###注文管理システムは何をしますか？

企業にとって、注文管理システムは、注文の入力から完了までの注文をデジタルで追跡する方法です。注文管理システムは、注文のライフサイクルを通じて発生するすべての情報とプロセスを記録します。これには、注文入力、経路、在庫管理、注文完了、および注文後のフォローアップ/サービスが含まれます。
