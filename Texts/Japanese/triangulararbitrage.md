# 三角裁定取引
##三角裁定取引とは何ですか？

三角裁定取引は、通貨の為替レートが正確に一致しない場合に発生する3つの外貨間の不一致の結果です。これらの機会はまれであり、それらを利用するトレーダーは通常、プロセスを自動化するための高度なコンピューター機器および/またはプログラムを持っています。

たとえば、三角裁定取引を採用しているトレーダーは、あるレート（EUR / USD）で金額を交換し、再度変換（EUR / GBP）してから、最終的に元のレート（USD / GBP）に変換し、[トランザクションが少ないと想定します。](/transactioncosts)[コスト](/transactioncosts)、純利益。

##三角裁定取引を理解する

このタイプの[裁定取引は、相場の為替レートが市場の](/arbitrage)[相互為替レート](/crossrate)と等しくない場合、「リスクのない」利益をもたらす可能性があります。言い換えれば、2つの通貨が3番目の通貨に対しても取引される場合、3つすべての為替レートを同期させる必要があります。そうでない場合、利益の機会が存在します。

通貨でマーケットメイクを行う国際銀行は、ある市場が過大評価され、別の市場が過小評価されている市場の非効率性を利用しています。為替レート間の価格差はほんの数セントであり、この形式の裁定取引が利益を生むためには、トレーダーは大量の資本を取引する必要があります。

##自動取引プラットフォームと三角裁定取引

自動取引プラットフォームは、特定の基準が満たされると取引が自動的に実行されるアルゴリズムが作成されるため、取引の実行方法を合理化しました。自動取引プラットフォームにより、トレーダーは取引を開始および終了するためのルールを設定でき、コンピューターはそのルールに従って取引を自動的に実行します。自動取引には、資本を危険にさらす前に履歴データの一連のルールをテストする機能など、多くの利点がありますが、三角裁定取引を行う機能は、自動取引プラットフォームを使用した場合にのみ実現可能です。

市場は本質的に自己修正エンティティであるため、取引は非常に速いペースで行われるため、裁定取引の機会は出現してから数秒で消滅します。自動取引プラットフォームは、機会を特定し、それが消える前にそれに基づいて行動するように設定できます。

とは言うものの、アルゴリズム取引プラットフォームと市場のスピードもトレーダーに逆行する可能性があります。たとえば、トレーダーが数秒で利益を上げる前に収益性の高い価格を固定できないという執行リスクが存在する可能性があります。

##三角裁定取引の例

例として、100万ドルがあり、次の為替レートが提供されているとします：EUR / USD = 1.1586、EUR / GBP = 1.4600、およびUSD / GBP=1.6939。

これらの為替レートでは、仲裁の機会があります。

1.ドルを売ってユーロを買う：100万ドル÷1.1586=€863,110

1.ユーロをポンドで売る：€863,100÷1.4600=£591,171

1.ポンドをドルで売る：£591,171x 1.6939 = $ 1,001,384

1.最終金額から初期投資を差し引きます：$ 1,001,384 – $ 1,000,000 = $ 1,384

これらの取引から、1,384ドルの裁定取引利益を受け取ります（取引費用や税金がないと仮定した場合）。

##ハイライト

-三角裁定取引は、アルゴリズム取引を通じて為替レートの不一致を利用する、通貨トレーダーによる低リスクの利益創出の一形態です。

-利益を確保するために、そのような取引は迅速に実行され、規模が大きくなければなりません。

-三角裁定取引の機会が定期的に利用されるため、通貨市場はより効率的になります。
