# 承認コード
##認証コードとは何ですか？

認証コードは、ユーザーがアイテムを購入、販売、転送したり、セキュリティで保護されたスペースに情報を入力したりすることを許可する英数字のパスワードです。認証コードは通常、文字、数字、または両方の組み合わせのシーケンスであり、個人のIDを検証したり、トランザクションを承認したり、セキュリティで保護された領域へのアクセスを提供したりします。

[クレジットカード発行者](/creditcard)から販売者に送信され、顧客のクレジットカードに取引を承認するのに十分なクレジットがあることを確認するためのものです。

>異なるプラットフォームは、ユーザーにパスワードの作成を求めるときに特定の要件を課す場合があります。たとえば、句読点（大文字と小文字の両方）または最小の長さのしきい値を使用する必要がある場合があります。

>>

##認証コードを理解する

認証コードは、ユーザーがアクセスできる制限があるトランザクションまたはエントリに使用されます。たとえば、クレジットカード認証コードは、発行銀行から販売を認証するベンダーへの5つまたは6つの番号のコードです。使用したクレジットカードが偽造品である場合、またはカードが所定の限度額を超えている場合、クレジットカード会社は自動的に販売を拒否します。承認されると、認証コードがクレジットカード取引に添付されます。これは、取引が合法であることを販売者に通知すると同時に、商品の返品や購入紛争などのフォローアップ検査で取引を特定するのに役立ちます。

認証コードはデジタルで送信され、[クレジットカードの処理を高速化するために使用されます](/batch-credit-card-processing)。ベンダーが発行者に口頭での認証コードを要求する必要がある場合、すべてのトランザクションを完了するために、商取引の速度が大幅に低下します。

##経費承認とデータセキュリティの承認コード

[企業の財務管理](/internalcontrols)においても役割を果たします。スタッフには、購入と経費について異なる認証コードが提供される場合があります。これにより、企業は特定の領域での購入と支出を従業員レベルまで追跡できます。

これらの認証コードには、特定のトランザクションしきい値も与えられています。従業員が自分の許容しきい値を超えて何かを支出しようとしている場合は、マネージャー、スーパーバイザー、または組織階層の上位にある別のスタッフが承認コードを提供する必要があります。この意味で、認証コードは、従業員の不正や資金の不正流用と戦うために使用できる統合的な制御メカニズムです。

認可コードは、情報セキュリティを維持するために専門的な職場でも一般的に使用されるようになりました。サーバーまたはVPNへのアクセスは、機密データベースへのアクセスを許可されるユーザーを制御するために、一意のユーザーIDに関連付けられた認証コードを使用して管理できます。

これらの認証コードは、従業員の在職期間中は永続的に使用される場合がありますが、パスワードコントロールの更新と同様に、定期的に更新されることがよくあります。 1回限りの認証コードまたはトークンもあり、1回のセッションの長さだけ持続します。

##ハイライト

-**認証コードは、ユーザーがさまざまなトランザクションを実行することを認証する英数字のパスワードです。**

-**企業の世界では、特定のトランザクションしきい値を含む購入と経費について、スタッフにさまざまな認証コードを与えることができます。**

-**認証コードは、従業員の在職期間中は永続的に使用される場合がありますが、定期的な更新が必要になることがよくあります。**
