# テイクプロフィットオーダー-T/P
##テイクプロフィットオーダー（T / P）とは

利益のためにオープンポジションをクローズする正確な価格を指定する一種の[指値注文です。](/limitorder)証券の価格が制限価格に達しない場合、テイクプロフィット注文は履行されません。

##テイクプロフィット注文の基本

[ストップロス注文](/stop-lossorder)（S / L）と組み合わせてテイクプロフィット注文を使用して、オープンポジションを管理します。証券がテイクプロフィットポイントまで上昇すると、T / P注文が実行され、ポジションは利益のためにクローズされます。セキュリティがストップロスポイントに落ちた場合、S / Lオーダーが実行され、ポジションは損失のためにクローズされます。市場価格とこれらの2つのポイントの違いは、取引の[リスク対](/riskrewardratio)[報酬の比率を定義するのに役立ちます](/riskrewardratio)。

テイクプロフィット注文を使用する利点は、トレーダーが手動で取引を実行したり、自分で推測したりすることを心配する必要がないことです。一方、テイクプロフィットの注文は、[基礎となるセキュリティの](/underlying-security)動作に関係なく、可能な限り最良の価格で実行されます。在庫はより高くブレイクアウトし始める可能性がありますが、T / P注文はブレイクアウトの最初に実行される可能性があり、その結果、高い[機会コストが発生します](/opportunitycost)。

テイクプロフィット注文は、リスクの管理に関心のある短期トレーダーが最もよく使用します。これは、計画された利益目標が達成されるとすぐに取引から抜け出すことができ、将来の市場の低迷のリスクを冒さないためです。長期的な戦略を持つトレーダーは、利益を減らすため、そのような注文を支持しません。

テイクプロフィットの注文は、チャートパターン分析、サポートおよびレジスタンスレベルなど、他の形式の技術分析によって定義されたレベルで、または[ケリー基準などの資金管理手法を使用して行われることがよくあり](/kellycriterion)ます。多くのトレーディングシステム開発者は、自動トレードを行う際にテイクプロフィット注文を使用します。これは、それらが明確に定義され、優れた[リスク管理](/riskmanagement)手法として機能するためです。

##テイクプロフィット注文の例

トレーダーが[上向きの三角](/ascendingtriangle)チャートパターンを見つけて、新しい[ロングポジションを開いたとします](/long)。株価が急上昇した場合、トレーダーは現在の水準から15％まで上昇すると予想しています。株がブレイクアウトしない場合、トレーダーはすぐにポジションを終了し、次の機会に移動したいと考えています。トレーダーは、株価がそのレベルに達したときに自動的に売るために、市場価格より15％高いテイクプロフィット注文を作成する場合があります。同時に、彼らは現在の[市場価格より5パーセント低いストップロス注文を出すかもしれません](/market-price)。

テイクプロフィットとストップロスの順序の組み合わせにより、5：15のリスク対リワードの比率が作成されます。これは、各結果に到達するオッズが等しい場合、またはオッズがブレイクアウトシナリオに偏っている場合に有利です。

[ブレイクアウト](/breakout)後に在庫がどれだけ高くなるかについて自分自身を推測したりする必要がありません。明確に定義されたリスク対報酬の比率があり、トレーダーは取引が発生する前に何を期待するかを知っています。

##ハイライト

-テイクプロフィット注文は、セキュリティコストの急激な上昇から利益を得ることに関心のある短期トレーダーにとって有益です。

-T / P注文の制限価格は、基本分析または技術分析のいずれかを使用して行われます。

-Take-profit（T / P）注文は、指定された利益レベルに達したときにクローズされる指値注文です。
