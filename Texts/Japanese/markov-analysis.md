# マルコフ分析
##マルコフ分析とは何ですか？

マルコフ分析は、予測値が現在の状態によってのみ影響を受け、以前のアクティビティによっては影響を受けない変数の値を予測するために使用される方法です。本質的には、変数を取り巻く現在の状況のみに基づいてランダム変数を予測します。

マルコフ分析は、大勢の人々の行動や決定を予測するためによく使用されます。これは、偶然の操作を伴うプロセスである確率的プロセスの研究を開拓したロシアの数学者アンドレイ・アンドレイエビッチ・マルコフにちなんで名付けられました。マルコフは最初にこの方法を適用して、コンテナに閉じ込められたガス粒子の動きを予測しました。

##マルコフ分析を理解する

マルコフ分析プロセスには、変数の現在の状態を前提として、将来のアクションの可能性を定義することが含まれます。各状態での将来のアクションの確率が決定されると、決定木を描画し、結果の可能性を計算できます。

マルコフ分析は、ビジネスの世界でいくつかの実用的なアプリケーションがあります。これは、ライン上のマシンの動作ステータスを考慮して、アセンブリラインから外れる不良部品の数を予測するためによく使用されます。また、貸倒れとなる企業の[売掛金（AR）](/accountsreceivable)の割合を予測するためにも使用できます。

企業は、マルコフ分析を使用して、現在の顧客の将来のブランドロイヤルティと、企業の[市場シェアに対するこれらの消費者の決定の結果を予測することもできます](/marketshare)。一部の株価およびオプション価格の予測方法には、マルコフ分析も組み込まれています。

##マルコフ分析の長所と短所

マルコフ分析の主な利点は、単純さとサンプル外の予測精度です。マルコフ分析に使用されるような単純なモデルは、より複雑なモデルよりも予測に優れていることがよくあります。この結果は、[計量経済学](/econometrics)でよく知られています。

残念ながら、マルコフ分析はイベントの説明にはあまり役立ちません。また、ほとんどの場合、根本的な状況の真のモデルにはなりません。はい、現在の状態に基づいて[条件付き確率](/conditional_probability)を推定するのは比較的簡単です。しかし、それはしばしば何かが起こった理由について少し教えてくれます。

>マルコフ分析は予測を行うための貴重なツールですが、説明は提供していません。

>>

エンジニアリングでは、マシンが故障する可能性を知っていても、マシンが故障した理由を説明できないことは明らかです。さらに重要なことに、マシンは、今日故障したかどうかの関数である確率に基づいて実際には故障しません。実際には、ギアをより頻繁に潤滑する必要があるため、機械が故障する可能性があります。

[金融市場](/financial-market)に関する知識が比較的不足しているために複雑になります。マルコフ分析は、そもそも[信用不良リスク](/creditrisk)を選別するよりも、債務不履行になる部分を推定するのにはるかに役立ちます。

##マルコフ分析の例

マルコフ分析は、株式投機家が使用できます。勢いのある[投資家](/momentum_investing)が、お気に入りの株が今日市場を打ち負かす可能性が60％であると推定するとします。この推定には現在の状態のみが含まれるため、マルコフ分析の重要な制限を満たしています。

マルコフ分析により、投機家は、株式が今日の市場を上回った場合、今後2日間の両方で株式が市場をアウトパフォームする確率は0.6 * 0.6 = 0.36または36％であると推定することもできます。[レバレッジ](/leverage)と[ピラミッド](/pyramiding)を使用することにより、投機家はこのタイプのマルコフ分析からの潜在的な利益を増幅しようとします。
