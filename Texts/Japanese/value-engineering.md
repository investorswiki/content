# バリューエンジニアリング
##バリューエンジニアリングとは何ですか？

バリューエンジニアリングは、プロジェクトで必要な機能を最小のコストで提供するための体系的で組織化されたアプローチです。バリューエンジニアリングは、機能を犠牲にすることなく、より安価な代替品で材料と方法を置き換えることを促進します。物理的な属性ではなく、さまざまなコンポーネントや材料の機能にのみ焦点を当てています。バリューエンジニアリングは、バリュー分析とも呼ばれます。

##バリューエンジニアリングを理解する

バリューエンジニアリングとは、設計段階で新規または既存の製品をレビューして、コストを削減し、機能を向上させて製品の価値を高めることです。アイテムの価値は、その目的を損なうことなくアイテムを生産する最も費用効果の高い方法として定義されます。したがって、品質を犠牲にしてコストを削減することは、単に[コスト削減](/cost-cutting)戦略です。

>バリューエンジニアリングでは、コスト削減が開発または分析される製品の品質に影響を与えることはありません。

>>

バリューエンジニアリングの概念は、第二次世界大戦の真っ只中にあるゼネラルエレクトリックで1940年代に発展しました。戦争のため、購入エンジニアのローレンスマイルズなどは、慢性的な不足があったため、材料や部品の代替品を探しました。これらの代替品は、多くの場合、コストを削減し、同等以上のパフォーマンスを提供することがわかりました。

###特別な考慮事項

マイルは、製品の価値を2つの要素の比率として定義しました：機能とコスト。アイテムの機能は、実行するように設計された特定の作業であり、コストとは、アイテムの[ライフサイクル](/product-life-cycle-management)中のコストを指します。機能とコストの比率は、機能を向上させるか、コストを下げることで、製品の価値を高めることができることを意味します。バリューエンジニアリングでは、生産、設計、保守、交換に関連するコストが分析に含まれます。

たとえば、新しいハイテク製品が設計されており、ライフサイクルがわずか2年になる予定であるとします。したがって、製品は、製品のライフサイクルの終わりまで役立つ最も安価な材料とリソースを使用して設計され、製造業者と[最終消費者の](/end-user)お金を節約します。これは、コストを削減して価値を向上させる例です。

別の製造会社は、最小のコストで製品の機能を最大化することによって付加価値を生み出すことを決定するかもしれません。この場合、製品の目的の詳細な分析を開発するために、アイテムのすべてのコンポーネントの機能が評価されます。価値分析の一部では、プロジェクトまたは製品がその機能を達成できる複数の代替方法を評価する必要があります。

リストされているさまざまな方法は、プロジェクトに実装できるいくつかの基本的および二次的な実行可能なオプションに絞り込まれています。例えば、石鹸の一部が側面に漏れた後に滑りやすくなる食器用洗剤のボトルは、グリップを改善し、漏れを最小限に抑えるためにボトルと開口部の注ぎ口の形状を再設計することによって改善される可能性があります。この改善により、追加の広告費用をかけずに売上を伸ばすことができます。

##ハイライト

-バリューエンジニアリングは、機能を犠牲にすることなく、より安価な代替品で材料と方法を置き換えることを促進します。

-物理的な属性ではなく、さまざまなコンポーネントや材料の機能にのみ焦点を当てています。

-バリューエンジニアリングは、プロジェクトに必要な機能を最小のコストで提供するための体系的で組織化されたアプローチです。
