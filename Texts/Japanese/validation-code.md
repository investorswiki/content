# 検証コード
##検証コードとは何ですか？

[クレジットカード](/creditcard)の前面または背面にある一連の3つまたは4つの番号です。これは、オンラインまたは電話で行われるクレジットカード取引に追加のセキュリティ層を提供することを目的としています。

ほとんどのクレジットカード発行者は、カードの裏面、署名パネルの右端に検証コードを配置します。ただし、American Express（AXP）カードでは、検証コードがカードの前面に印刷されています。

##検証コードのしくみ

オンラインショッピングの人気が高まるにつれ、個人情報の盗難やその他の形態のクレジットカード詐欺の脅威はますます深刻になっています。このリスクを軽減するために取られる対策の1つは、クレジットカードを購入する際に検証コードを使用することです。

通常の取引では、顧客は名前、請求先住所、カード番号、有効期限、および検証コードを提供するように求められます。名前や住所など、これらの詳細の多くは他のソースから入手できます。カード番号、有効期限、および検証コードは、理論的にはカード自体を所有している場合にのみ取得できます。追加の手段として、検証コードは通常、カードの裏面に印刷されているため、泥棒になる可能性のある人がクレジットカードの1枚の写真から必要なすべての情報を収集することはより困難になります。

これらのセキュリティ対策をさらに強化するために、消費者保護法により、購入後に販売者が顧客の検証コードを保存することは禁止されています。ただし、悪意のある販売者はこの情報を違法に記録する可能性があります。追加の保護手段は、[ポイントオブセール（POS）](/point-of-sale)端末を使用して支払いを行うときにカード所有者が入力する必要のある個人識別番号（PIN）によって提供されます。

##検証コードの実際の例

検証コードなどのセキュリティ対策により、身元の盗難や盗まれたクレジットカードを使用した購入が困難になりますが、十分な意欲のある泥棒を阻止することはできません。実際には、クレジットカード詐欺は近年増加を続けており、2020年に報告された393,207件を超えています。米国は世界で最も深刻な影響を受けている国であり、世界の件数の34％近くを占めています。

顧客が購入した後、加盟店はカードのセキュリティコードを保存できません。これにより、クレジットカードの盗難に対する保護が強化されます。それでも、検証コードは盗まれる可能性があり、カード所有者は、カード番号と有効期限を保護するのと同じように、カードの検証コードを保護する必要があります。検証コードは、泥棒が他人のカードと不正な取引を行うことを可能にする重要なデータです。

ただし、泥棒が盗まれたカードを使用した場合、カード所有者の責任は、盗難が報告された時期に応じて、公正信用請求法（FCBA）に基づいて50ドルに制限されます。カードが紛失していることに気付いた場合、または疑わしい購入や不正な購入などの活動を検出した場合は、すぐにクレジットカードの発行者に連絡して問題を報告し、詐欺の可能性があることを警告する必要があります。その後、カード発行者はカードをキャンセルまたは非アクティブ化できます。

##ハイライト

-クレジットカードの表面または裏面に印刷された3文字または4文字のコードで構成されています。

-検証コードは、クレジットカード詐欺を減らすために導入されたセキュリティ対策の1つです。

-最新のNilsonReportによると、クレジットカード詐欺の事例は増え続けており、2019年には290億ドル近くに達し、2027年までに約380億ドルに達すると予測されており、最近報告されたものの大部分を米国が占めています。ほぼ34％の損失。
