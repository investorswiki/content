# 見積もり依頼（RFQ）
##見積もり依頼（RFQ）とは何ですか？

見積もり依頼（RFQ）は、入札案内（IFB）とも呼ばれ、特定のタスクまたはプロジェクトを遂行する機会を得るために、企業が選択したサプライヤーおよび請負業者に価格見積もりと入札を提出するように求めるプロセスです。 RFQプロセスは、特定の数の標準製品の一貫した供給を必要とする企業にとって特に重要です。企業は、RFQを単独で送信することも、提案要求（RFP）の前に送信することもできます。

##見積もりのリクエストはどのように機能しますか

RFQは通常、[提案依頼書（RFP）を送信するための最初のステップ](/request-for-proposal)です。これらの2つのドキュメントは、必要なプロジェクトまたはサービスの詳細を提供するという点で類似していますが、RFQは通常、より包括的な価格見積もりを求めます。また、企業は通常、必要な数量がわかっているジェネリック製品のRFQを設計し、RFPは、数量と仕様が不明な独自のニッチなプロジェクト向けです。

価格設定に加えて、RFQには、支払い条件、企業の入札選択に影響を与える可能性のある要因、提出期限などの詳細が含まれる場合があります。たとえば、特定のハードドライブサイズと処理速度を備えた500台のコンピューターを購入したい[政府機関は、入札予定者として複数のベンダーにRFQを送信します。](/federal-agencies)

RFQ形式は特定の企業内で統一されているため、RFQが見積もりとともに戻ってきた場合、勧誘企業はそれらを簡単に比較できます。通常、RFQプロセスは、準備フェーズ、処理フェーズ、授与フェーズ、および終了フェーズの4つのセクションで構成されます。会社は通常、最低資格基準を満たし、最低入札額を提示するベンダーに契約を授与します。

##特別な考慮事項

RFQは公の発表ではありません。勧誘会社は信頼できる企業にのみRFQを送信するため、長い[調達](/procurement)文書を作成する必要はありません。また、公募とは異なり、企業は要求した数の入札のみを取り戻すことができるため、時間の節約にもなります。

RFQを使用すると、商品やサービスの調達に必要な時間が短縮されます。また、企業は希望するベンダーからのみ入札を受け取るため、ある程度のセキュリティも提供されます。一方、RFQは競争の量を減らすため、企業は入手可能な最低価格を受け取ったり、新しい高品質のベンダーについて学んだりすることを見逃す可能性があります。

企業がRFQに応じて見積もりを受け取った場合、それはオファーでも拘束力のある契約でもありません。弁護士は、発注書を送信することにより、選択したベンダーに仕事を提供します。これは、事実上、作業の条件を指定する契約です。ベンダーが発注書を受け入れて署名すると、契約が開始されます。

##ハイライト

-企業は通常、標準製品の数量がわかっていて、ニーズが継続しているときにRFQを送信します。

-RFQは、単独で送信することも、提案依頼書（RFP）と組み合わせて送信することもできます。

-企業は特定のベンダーや請負業者を対象としているため、RFQは一方的な入札や見積もりを生成しません。

-見積もり依頼（RFQ）は、特定のタスクまたはプロジェクトについて、企業が選択したサプライヤーおよび請負業者から見積もりを求めるビジネスプロセスです。
