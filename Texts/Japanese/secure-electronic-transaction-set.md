# Secure Electronic Transaction（SET）
## Secure Electronic Transaction（SET）とは何ですか？

Secure Electronic Transaction（SET）は、電子商取引のWebサイトで電子デビットカードとクレジットカードの支払いを保護するために使用されていた初期の通信プロトコルでした。安全な電子トランザクションは、インターネット上の電子ポータルを介した消費者カード情報の安全な送信を容易にするために使用されました。安全な電子トランザクションプロトコルは、カード情報の個人情報をブロックし、商人、ハッカー、電子泥棒が消費者情報にアクセスするのを防ぐ役割を果たしました。

## Secure Electronic Transaction（SET）を理解する

安全な電子取引プロトコルは、VisaやMasterCardなどの電子取引の主要プロバイダーのほとんどによってサポートされていました。これらのプロトコルにより、マーチャントは実際に見ることなく顧客のカード情報を確認できるため、顧客を保護できます。カードの情報は、確認のためにクレジットカード会社に直接転送されました。

安全な電子取引のプロセスでは、クレジットラインであろうと銀行口座であろうと、資金への電子アクセスを提供するために割り当てられたデジタル証明書を使用しました。購入が電子的に行われるたびに、トランザクションの参加者（顧客、商人、金融機関）に対して暗号化されたデジタル証明書が生成され、一致するデジタルキーにより、相手の証明書を確認してトランザクションを検証できるようになりました。使用されるアルゴリズムは、対応するデジタルキーを持つ当事者のみがトランザクションを確認できることを保証します。その結果、消費者のクレジットカードまたは銀行口座情報を使用して、口座番号などの個人情報を明かさずに取引を完了することができます。安全な電子取引は、アカウントの盗難、ハッキング、およびその他の犯罪行為に対するセキュリティの一形態であることが意図されていました。

##安全な電子取引の歴史

安全な電子トランザクションプロトコルの開発は、eコマーストランザクション、特にインターネットを介した消費者主導の購入の出現と成長への対応でした。オンラインでビジネスを行うことは、1990年代半ばの新しい現象でした。同様に、これらのトランザクションを保護するために利用できるセキュリティはまだ開発中であり、さまざまな程度で効果的でした。安全な電子取引標準によって定義されたプロトコルにより、小売業者や金融機関は、デジタル取引を適切に復号化して処理するための適切なソフトウェアを備えていたため、オンライン決済システムを使用できました。 1996年、SETコンソーシアム（VISAとMastercardがGTE、IBM、Microsoft、Netscape、SAIC、Terisa Systems、RSA、VeriSignと協力して構成されたグループ）は、互換性のないセキュリティプロトコル（VisaとMicrosoftのSTT）を組み合わせるという目標を設定しました。 ; MastercardおよびIBMからのSEPP）を単一の標準に。

安全な電子取引によって定義されたプロトコルが導入された後、オンラインデビットおよびクレジットカード取引のデジタルセキュリティに関する他の標準が登場しました。安全な電子取引の初期の支持者の1人であるVisaは、顧客の安全なデジタル決済と取引のフレームワークとして、最終的に3Dセキュアと呼ばれる別のプロトコルを採用しました。 3Dセキュア方式は、拡張可能なマークアップ言語（XML）ベースのプロトコルであり、オンラインのクレジットカードおよびデビットカードのトランザクション用の追加のセキュリティレイヤーとして設計されています。

もともとはVisaとArcotSystems（現在はCA Technologiesとして知られています）によって共同執筆されました。 3Dセキュアに基づく同様のプロトコルが、Mastercard、Discover、およびAmericanExpressで使用されるようになりました。

##ハイライト

-1990年代半ばに安全な電子取引によって定義されたプロトコルが導入された後、オンラインデビットおよびクレジットカード取引のデジタルセキュリティに関するその他の標準が登場しました。

--Visaは、3-D Secureと呼ばれる新しい標準のセキュリティプロトコルを早期に採用し、最終的にはMastercard、Discover、AmericanExpressによってさまざまな形で採用されました。

-Secure Electronic Transactionは、1996年に開発され、電子商取引のWebサイトで電子デビットカードとクレジットカードの支払いを保護するために使用された初期の通信プロトコルでした。

-安全な電子トランザクションプロトコルにより、マーチャントは実際に見ることなく顧客のカード情報を確認できるため、アカウントの盗難、ハッキング、およびその他の犯罪行為から顧客を保護できます。
