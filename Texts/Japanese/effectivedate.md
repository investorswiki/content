# 発効日
##発効日とは何ですか？

契約法では、発効日は、署名者間または署名者間の合意または取引が拘束力を持つようになる日です。[初期公募（IPO）の](/ipo)場合、これは株式が取引所で最初に取引できる日付です。

>契約の両当事者は通常、契約に署名する前に発効日に合意する必要があります。

>>

##発効日を理解する

ビジネス契約および取引は、契約の当事者が契約に基づいて履行する義務を開始する時期である発効日とともに文書化されます。これらの契約は、雇用契約、クレジットまたはローン契約、または商取引の形をとることができます。発効する「日付」に関する限り、当事者は、契約を正式に開始するのは署名日、すでに経過した日（[バックデート](/backdating)）、または将来の日付にするかどうかを決定します。

公開を希望する企業の場合、発効日は証券取引委員会（SEC）に証券が登録されてから30日以内に発生し、SECが[フォームS-1](/sec-form-s-1)の登録を確認して開示の完全性を確認する時間を与えます。将来の投資家は情報に基づいた決定を下すことができます。このレビュー期間中、SECは質問をしたり、説明を要求したり、ファイリングの特定のセクションに記入または修正するように会社に指示したりすることができます。

##発効日の例

IPOプロセスは、SECによって厳密に管理されています。 Twilio Inc. 2016年5月26日にIPOを申請しました。その直後、会社は目論見書の表紙に次のように印刷するように注意しながら、修正された申請書を提出しました。

>>

>登録者は、この登録声明がその後、のセクション8（a）に従って発効することを具体的に述べるさらなる修正を提出するまで、発効日を延期するために必要となる可能性のある日付にこの登録声明を修正します。 1933年の証券法、または登録声明が発効するまでは、証券取引委員会が第8条（a）に従って行動することを決定する場合があります。

>>

発効日は2016年6月23日でした。Twilioの株式はその日から取引を開始しました。

ウェブサイトでは、発効日は利用規約とプライバシーポリシーに記載されていることがよくあります。企業が非常に専門的なエンドユーザーまたはエンドユーザーのグループと個別の契約を結ぶために特別な努力をしない限り、利用規約（または利用規約）とプライバシーポリシーが特定のウェブサイトまたはインターネットのすべてのユーザーに適用されますプラットホーム。

ユーザーは通常、関連するアプリをダウンロードするか、Webサイトにログオンするときに、利用規約に同意する必要があります。これらの用語は、それらの用語が更新され、すべてのユーザーが改訂された用語を承認するように求められない限り、他のときに与えられ、他の用途で読まれる用語と異なってはなりません。

このような状況では、利用規約とプライバシーポリシー契約の発効日は、ユーザーが同意したときではなく、利用規約が最後に更新されたときです。これらのタイプの契約の場合、これらの日付は「発効日」ではなく、「最終改訂」または「最終更新」で示されます。

##ハイライト

-発効日は、過去（過去）または将来の日付にすることができます。

-発効日とは、契約の当事者が契約に基づいて履行する義務を開始する時期です。
