# 取引
##トランザクションとは何ですか？

トランザクションとは、金銭と引き換えに商品、サービス、または金融資産を交換するための買い手と売り手の間の完全な合意です。

ビジネス簿記では、この「トランザクション」のわかりやすい定義は扱いにくい場合があります。取引は、[発生主義会計](/accrualaccounting)と[現金会計](/cashaccounting)のどちらを使用するかに応じて、会社によって早期に記録される場合と、後で記録される場合があります。

##トランザクションを理解する

買い手と売り手の間の販売取引は比較的簡単です。人物Aは、製品またはサービスと引き換えに人物Bに支払います。彼らが条件に同意すると、お金は商品またはサービスと交換され、取引は完了します。

、将来の日付まで[決済され](/cashsettlement)ない取引を行う可能性があるため、会計の世界では取引がより複雑になる可能性があります。または、既知であるがまだ支払期日が到来していない収益または費用がある場合があります。[サードパーティのトランザクション](/third-party-transaction)もプロセスを複雑にする可能性があります。

企業が発生主義会計または現金会計のどちらを使用して収益および費用の取引を記録するかは、会社の財務および税務報告に影響します。

-発生主義会計方式では、金銭の受け取りや費用の支払いに関係なく、発生時に取引を記録する必要があります。

-現金会計方式は、お金を受け取ったとき、または費用を支払ったときにのみ取引を記録します。これには、意向表明書または覚書が必要になる場合があります。

>発生主義会計は、過去3年間で平均2500万ドルを超える企業によって最も頻繁に使用されますが、現金会計は主に中小企業によって使用されます。

>>

##発生主義会計を使用した取引

発生主義会計を使用する場合、会社はサービスの完了時または商品の配送時に収益を記録します。会社の収入を会計処理するときに在庫が必要であり、会社の[総](/gross-receipts)[収入](/gross-receipts)が過去3年間で平均2500万ドルを超える場合、会社は通常、売上と購入の会計に発生主義を使用します。

###発生主義会計の例

たとえば、10月にストアクレジットで顧客に商品を販売する会社は、取引[を売掛金（AR）のアイテムとしてすぐに記録します](/accountsreceivable)。顧客が12月まで商品の現金支払いを行わなかったり、分割払いを行ったりした場合でも、取引は10月の収益として記録されます。

>顧客がクレジットで何かを購入した場合、その商品を販売する会社が発生主義会計方式を使用していれば、それはすぐに取引として記録されます。

>>

会社が購入する商品やサービスについても同じことが言えます。事業費は、製品またはサービスの受領時に記録されます。 4月にクレジットで購入した消耗品は、企業が5月まで消耗品の現金支払いを行わない場合でも、4月の費用として記録されます。

##現金会計を使用した取引

ほとんどの中小企業、特に[個人事業主](/soleproprietorship)と[パートナーシップ](/partnership)は、現金会計法を使用しています。収入は、顧客から現金、小切手、またはクレジットカードでの支払いを受け取ったときに記録されます。

###現金会計の例

ある企業が3月に10,000ドルのウィジェットを顧客に販売するとします。顧客は4月に請求書を支払います。同社は、4月に現金を受け取った後にのみ売却を認識します。

一方、費用は支払いが行われたときにのみ記録されます。たとえば、企業は5月に500ドルの事務用品を購入し、6月にそれらの支払いを行う場合があります。企業は、6月に請求書を支払うときに購入を認識します。

税務上の理由から、会計の現金主義は、企業の年間売上高が過去3年間で平均2500万ドル未満の場合にのみ利用できます。発生主義や[繰延](/accruals)などの複雑な会計取引が必要ないため、現金主義は取引を記録するための発生主義よりも簡単です。その欠点は、少なくとも紙面では、ビジネスの利益が月ごとに大きく異なる可能性があることです。

##ハイライト

-トランザクションには、商品またはサービスの金銭的交換が含まれます。

-発生主義会計は、支払いの受領または支払いの時期に関係なく、取引が完了した直後に取引を認識します。

-対照的に、主に中小企業で使用される現金会計は、お金を受け取ったり支払ったりしたときにのみトランザクションを記録します。
