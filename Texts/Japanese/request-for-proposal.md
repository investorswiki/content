# 提案依頼書（RFP）
##提案依頼（RFP）とは何ですか？

提案依頼（RFP）は、プロジェクトを発表し、それを説明し、資格のある請負業者にプロジェクトを完了するための入札を求めるビジネスドキュメントです。ほとんどの組織はRFPを使用してプロジェクトを立ち上げることを好み、多くの政府は常にRFPを使用しています。

RFPを使用する場合、入札を要求するエンティティは、提出された入札の実現可能性、入札会社の財務状態、および各[入札者](/bidder)のプロジェクトへの取り組み能力を評価する責任があります。

##提案要求（RFP）を理解する

RFPは複雑なプロジェクトに使用され、多くの場合、多数の下請け業者が必要です。それらは、RFPを発行する組織、実施されているプロジェクトの範囲、およびエントリーを評価するための基準について説明します。また、入札プロセスと契約条件についても概説します。

リクエストには、落札者が実行するタスクと作業を完了するためのタイムラインを説明する作業ステートメントが含まれています。

RFPはまた、入札のフォーマットと提示方法に関する具体的なガイダンスとともに、提案の作成方法について入札者にアドバイスします。それらには通常、入札者が含める必要のある情報と目的の形式に関する指示が含まれています。

>提案は、請負業者の創造性を妨げるほど詳細であったり、請負業者が困惑したままになるほど曖昧であったりしてはなりません。

>>

ほとんどのRFPは、政府機関や公共部門の他の組織によって発行されます。彼らは一般的に、民間企業間の競争を開き、プロセスからバイアスを取り除くために必要とされます。代理店は、最低かつ最も競争力のある入札を確実に取得したいと考えています。

ただし、民間または公的機関は、プロジェクトに関する複数の入札とさまざまな視点を得るためにRFPを出すことができます。

たとえば、報告プロセスを紙ベースのシステムからコンピュータベースのシステムに変更したい企業は、新しいシステムを確立して統合するためのハードウェア、ソフトウェア、およびユーザートレーニングプログラムの提案を要求する場合があります。仕事。競争力のある入札プロセスにより、利用可能な代替案についてより深い洞察が得られる可能性があります。

##RFPの要件

政府機関またはその他の組織は、完全でオープンな競争を提供し、ソリューションのコストを削減するための提案依頼書を発行する必要がある場合があります。仕様に最も敏感な提案を受け入れることは、必ずしも最低価格の入札を意味するわけではありません。

提案依頼書を巧みに作成することで、結果として得られるソリューションの成功または失敗を確実にすることができます。指定された要件があいまいすぎる場合、入札者は問題の適切な解決策を設計および実装できない可能性があります。要件が詳細で制限的すぎる場合、入札者の革新は制限される可能性があります。

RFPプロセスは、提案依頼書の作成から始まります。入札者は勧誘を確認し、改善のための提案を提出します。[フィードバックを](/negative-feedback)実施した後、最終的な提案依頼書が発行されます。次に、入札者は提案を提出します。

顧客は、選択を入札者の小グループに絞り込み、価格設定と技術的な詳細について交渉を開始します。顧客は、契約を締結する前に、残りの入札者に最良かつ最終的なオファーを提出するように依頼することができます。その後、契約は会社に提示され、問題に対する最善の解決策を提供します。

##RFPのメリット

RFPは、部分的には広告です。プロジェクトが進行中であることを発表し、仕事を成し遂げることができる資格のある候補者への扉を開きます。

政府では、RFPは、契約の報奨の要因としてクロニズムを確実に排除する方法として採用されています。また、競争へのプロセスが開かれ、プロジェクトのコストを低く抑えることが期待できます。

RFPの代替手段は、プロジェクトマネージャーがプロジェクトの潜在的なベンダーを調査および特定する必要がある、あまり正式ではないプロセスです。検索がどれだけ網羅的であるかに応じて、潜在的な応答が制限される可能性があります。新しいベンダーや革新的な答えが明らかになる可能性は低くなります。

##RFP対RFQ対RFI

提案の要求（RFP）、見積もりの要求（RFQ）、および情報の要求（RFI）は、企業やその他のエンティティがサプライヤまたは請負業者のためにビジネスコミュニティに連絡するために使用する3つの異なるタイプのドキュメントです。一緒に働く。

-上記のように、提案依頼書は、計画されている特定のプロジェクトを発表し、仕事を成し遂げることができる請負業者を求めます。

-[見積依頼](/request-for-quote)（RFQ）は、特定の製品またはサービスを提供する契約の入札を求める多くのサプライヤーに送信される勧誘です。要求には、必要な質と量、および会社または組織が望むタイミングを指定する必要があります。

-情報の要求（RFI）は、サプライヤーが提供できる製品およびサービスに関する書面による情報をサプライヤーに求めることです。これは、将来の参照のためにサプライヤーのデータベースの情報を収集するために使用される可能性があります。

##RFPの例

たとえば、連邦鉄道局が、高速鉄道システムの資金調達、設計、建設、運用、および保守に関する提案の要求を発行したとします。

利害関係者は、文書に概説されている要件を満たす提案を提出します。期限までに受け取った提案に基づいて、運輸省は提案のさらなるレビューと開発のための委員会を設立します。

DOTは、その目標を最も網羅する提案を選択し、会社を雇用して作業を実行します。

###詳細なRFPを見る

チェサピークベイトラストのこのサンプルRFPは、リクエストを成功させるために必要な適切なレベルの詳細を示しています。提案では、プロジェクトについて説明し、組織の概要を示し、追求しているサービスの目標を定義し、提案がどのように評価されるかを説明します。予想される形式、予算の制限、および期限の要件を伝達します。

RFPには、信頼が何を求めているかをさらに明確にするために、多数のサポートドキュメントと展示も含まれています。これは、追求されているサービスの範囲を定義するのに役立つだけでなく、関心のある企業がスケジュール、ロジスティクス、および価格設定を評価するのに役立ちます。

その他のサンプルは、ハウツー情報のライブラリと非営利団体向けのサンプルRFPを提供するサイトであるTechSoupから見つけることができます。

##結論

RFPは、プロジェクトを発行する会社とそれに対応する会社のために、プロジェクトを定義します。適切に作成されたRFPは、提案の背後にある意図を伝え、最終結果が期待に応えることを保証します。

また、オープンプロセスを保証します。理想的には、複数の入札者が応答します。これにより、組織はさまざまなアプローチと価格を調査し、ニーズに最適なものを選択する機会が得られます。

### RFP要件とは何ですか？

RFPは、実行可能な応答を引き付けるためにプロジェクトを十分に詳細に記述および定義する必要があります。入札予定者は、ビジネスの性質とプロジェクトで達成したい目標を理解できる必要があります。プロジェクトは、入札者がその範囲と、プロジェクトを実行するために提供する必要のあるすべての製品とサービスを明確に理解できるように、十分に詳細に定義する必要があります。予想される提案の形式も詳細にする必要があります。オファーを比較対照するには、統一された応答が必要です。RFPはかなり厳格な形式に従いますが、その形式は、それらを準備する機関や企業によって異なる場合があります。ノースカロライナ州のRTIInternationalからのこのサンプルは、概要と背景、成果物の説明、および選択基準に関する情報を含む、典型的なRFPの要素を示しています。

### RFPとはどういう意味ですか？

提案要求（RFP）は、それを発行する会社または他の組織によって提案された新しいプロジェクトを完了するための入札の公開要求です。これは、競争を開き、プロジェクトの計画者が検討する可能性のあるさまざまな代替案を奨励することを目的としています。

### RFPとRFQの違いは何ですか？

ハードウェアストアが園芸用品部門を拡大しており、棚を埋めるために必要なサプライヤーを見つける必要があるとします。ガーデニング製品の潜在的なサプライヤーの数に見積もり要求（RFQ）を送信する場合があります。応答により、確立された価格で、必要な商品の量と質の1つ以上のサプライヤーとの関係を確立することができます。会社または他の組織がRFQを送信するとき、それは必要なものを正確に把握し、求めています。 RFPは、よりオープンエンドなプロセスです。企業または組織は、新しいプロジェクトを実行するための資格のある請負業者を探しています。プロジェクトとその目標はある程度詳細に定義されていますが、応答には創造性の余地があります。たとえば、子供用公園のRFPを使用すると、請負業者は公園に含まれる可能性のある正確なレイアウト、機器、および設備を提案できます。

### RFP後に何が起こりますか？

RFPの後にPが続きます。提案はレビューのために提出されます。プロジェクトの規模と範囲によっては、このレビューは多数の委員会が関与するマルチレベルのプロセスになる場合があります。特に政府機関は機敏に動くことで知られていませんが、時間がかかる場合でも、レビュープロセスを使用して、最終的な選択と開始日を視野に入れて追加情報の提出を求められる可能性のある少数のファイナリストに提案を絞り込みます。プロジェクトのために。

###いつRFPを使用しますか？

大小の企業は、現在の業務を維持するのに十分なリソースを持っている傾向があります。新しいプロジェクトを引き受けたい場合、それをワークロードに追加するためのリソースも専門知識も手元にありません。そのような場合、RFPは、プロジェクトを完了するために必要な専門知識を採用するための効率的な方法をビジネスに提供します。
