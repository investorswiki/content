# 一般公開ライセンス
GPL、またはGeneral Public License（GPL）は、ソフトウェアの無料のコピーレフトライセンスを指します。コンピュータプログラムやその他の種類の作品を自由に使用、読み取り、コピー、共有、変更、および配布する権利を誰にでも付与します。 GPLは元々GNUプロジェクト用に作成されたもので、一般的な使用に採用された最初のコピーレフトライセンスでした。これは、1989年の初めに、Free Software Foundation（FSF）の創設者であるRichardStallmanによって作成されました。

著作権とは対照的に、コピーレフトという用語は、GPLが派生作品の公開を許可しているが、元の作品と同じライセンス条件で配布する必要があることを意味します。したがって、ユーザーはGPLライセンスのソフトウェアを使用したり、別の種類のライセンスの下で派生物をリリースしたりすることはできません。これは、Berkley Software Distribution（BSD）やMITライセンスなど、他の種類の無料ソフトウェアライセンスには当てはまりません。

BSDおよびMITライセンスは、許容ライセンスのカテゴリに含まれます。コピーレフトライセンスとパーミッシブライセンスの両方で、ユーザーはソフトウェアをコピー、変更、および配布できますが、条件は多少異なります。

一方では、コピーレフトライセンスは、オープンソースソフトウェアが誰でも利用できることを保証します。また、無料で利用できるようになった作品から他の誰かが利益を得るのを防ぎます。コピーレフトの支持者は、自分たちの仕事に対するある程度の管理を維持することにもっと関心を持つ傾向があります。

一方、パーミッシブライセンスでは、元の開発者が参照され、作業に起因している限り、ソフトウェアを広く使用できます。言い換えれば、パーミッシブライセンスでは、あらゆる種類のライセンスの下で、誰でも作品をコピー、変更、および配布できます。唯一の要件は、元の作成者にクレジットを与えることです。

現在、GPLライセンスは、無料のオープンソースソフトウェアを配布するために最も広く使用されている方法です。 GNU Emacs、GNUコンパイラコレクション（GCC）、さらには[Linux](/linux)カーネルなどの人気のあるフリーソフトウェアプログラムは、このライセンスの下でライセンスされています。
