# プロセス値分析（PVA）
##プロセス値分析とは何ですか？

プロセス価値分析（PVA）は、合理化できるかどうかを判断するために企業が実施する内部プロセスの調査です。 PVAは、[顧客が何](/customer)を望んでいるかを調べ、その結果を達成するためにプロセスのステップが必要かどうかを尋ねます。 PVAの目標は、顧客の満足度を犠牲にすることなく、商品やサービスを作成するために必要な[バリューチェーン](/valuechain)で発生する不要な手順や費用を排除することです。その結果、商品やサービスをより迅速に、より低コストで顧客に提供できるようになります。

##プロセス値分析（PVA）を理解する

PVAを実施する際、管理者は、新しいテクノロジーを有益に実装できるかどうか、回避できるエラーが発生しているかどうか、プロセスに不要な追加のステップがあるかどうかなどを検討します。[経済的価値](/economic-value)を付加していないと特定されたプロセスのステップは、変更または破棄される可能性があります。プロセスをより効率的にする可能性のある新しいテクノロジーが出現すると、プロセスが繰り返し検討される場合があります。

> PVAでは、マネージャーが業務のすべての領域を客観的に分析し、付加価値がなく費用対効果の低い活動を特定する必要があります。

>>

経営陣はPVAを実施して、会社全体のあらゆる事業分野のプロセスをレビューおよび評価できます。たとえば、企業は、インバウンド[ロジスティクス](/logistics)、オペレーション、アウトバウンドロジスティクス、マーケティング、販売、およびカスタマーサービスのプロセスと機能を評価できます。

##プロセス価値分析（PVA）に対する批判

PVAのリスクの1つは、プロセスのいくつかの重要なステップが排除される可能性があることです。プロセスには、ルールが確実に守られるようにするためのコントロールポイントが含まれる場合があります。これらの規則は、[コスト管理を確立し](/cost-control)、適切な会計手順を保護し、その他の内部統制を保護するように設計されている場合があります。必要なコントロールポイントを削除すると、会社に意図しない結果が生じる可能性があります。

たとえば、PVAがコスト削減に重点を置きすぎると、企業はビジネスの円滑な運営を維持するプロセスを削除または変更するリスクを負います。これの例は、顧客サービス部門をサードパーティベンダーに[アウトソーシングすることを決定した会社で](/outsourcing)あり、ベンダーには優れた顧客サービスを提供するための労働力や専門知識がないことがわかりました。このような状況の発生を防ぐために、企業はPVAの監督を支援し、内部の会計スタッフおよび経営陣に相談するために統制アナリストを雇う場合があります。

##プロセス値分析（PVA）の例

一部の企業は、購入プロセスを合理化するためにPVAを採用しています。少額の購入の場合、彼らは主要なクレジットカード会社からマネージャー[調達カードを発行することを選択しました。](/procurement)これは、小規模な購入が大規模な購入に通常必要とされる複数のステップのプロセスを経ることを要求するよりも安価であることが証明されています。

[買収](/acquisition)を行ったときにPVAを実施することがあります。 PVAは、被買収企業のプロセスが買収企業のプロセスよりも効率的でないか、またはその逆であるかを明らかにする場合があります。 PVAは、経営陣が企業の統合を通じて達成したいと考えている[相乗効果または潜在的な経済的利益](/synergy)を最大化するのにも役立ちます。

##ハイライト

-PVAを実施することの欠点は、会社がプロセスを削除したり、ステップを変更したりして、商品の配送が遅れたり、顧客との関係が損なわれたりするなど、意図しない結果が生じる可能性があることです。

-PVAを実施する企業は、より低コストでより迅速に商品やサービスを顧客に提供したいと考えています。

-プロセス価値分析（PVA）は、特定のビジネスプロセスの各ステップを調べて、顧客の満足度を維持しながら、それを改善または合理化できるかどうかを判断します。
