# Метод замены цепи
## Что такое метод замены цепи?

Метод цепочки замещения представляет собой модель принятия решений по [бюджетированию капиталовложений](/capitalbudgeting),. которая сравнивает два или более [взаимоисключающих](/mutuallyexclusive) предложения капиталовложений с неравным сроком службы. Метод замещающих цепочек учитывает разные сроки жизни альтернативных планов, а также их ожидаемые денежные потоки. Это облегчает сравнение предложений.

При анализе цепочки замещения чистая приведенная стоимость ( [NPV](/npv) ) определяется для каждого плана. Можно выполнить одну или несколько итераций («звеньев» в цепочке замены), чтобы создать сопоставимые временные рамки для проектов. При сравнении предложений за одинаковые периоды времени информация о принятии-отклонении для различных проектов становится более надежной.

## Понимание метода цепочки замещения

Методология включает определение количества лет [денежного потока](/cashflow) (жизни проекта) для каждого из проектов и создание «цепочек замещения» или итераций для заполнения пробелов в менее продолжительном проекте. Предположим, что срок жизни проекта А составляет пять лет, а срок жизни проекта В — десять лет. Данные проекта А можно спроецировать на следующие пять лет, чтобы они соответствовали десятилетнему сроку жизни проекта Б. Конечно, любые чистые инвестиции и чистые денежные потоки для каждой итерации также принимаются во внимание. Затем можно рассчитать NPV каждого проекта, чтобы получить надежную информацию о принятии-отказе. Чистая приведенная стоимость представляет собой приведенную стоимость чистого денежного потока, полученного в результате проекта, дисконтированного по [стоимости капитала фирмы за](/costofcapital) [вычетом чистых инвестиций](/netinvestment) в проект .

Примеры типов проектов, в которых может быть полезен анализ методом цепочки замены, включают транспортную компанию, взвешивающую, стоит ли обновлять свой парк. Другой случай, когда он может быть использован, — это помощь горнодобывающей компании в оценке того, какой проект развития завода следует реализовать.

## Требования метода цепочки замещения

Не всегда возможно использовать метод цепочки замещения для сравнения проектов. Метод замещающей цепи требует повторяющихся проектов и постоянной ставки дисконтирования.

### Повторяемость

Во многих случаях можно выполнить более короткий проект несколько раз, как того требует метод цепочки замещения. Например, фирме, возможно, придется выбирать между ежемесячной арендой офисных помещений в текущем местоположении и [арендой](/lease) офисных помещений на один год в новом месте. Можно оценить проекты, используя метод цепочки замещения, сравнив чистые инвестиции и чистые денежные потоки за 12 одномесячных итераций аренды в текущем местоположении с арендой на один год в предлагаемом новом месте.

В других случаях метод цепочки замещения не может быть использован, поскольку проекты не могут быть повторены. Фирме, возможно, придется выбирать между обновлением своих старых компьютеров или покупкой новых систем. Новые системы прослужат дольше и будут стоить дороже, однако часто невозможно обновлять старые компьютеры несколько раз. Старые компьютеры могут иметь наилучшие процессоры, поддерживаемые их материнскими платами после обновления, поэтому их нельзя обновить снова.

### Постоянная ставка дисконтирования

В некоторых случаях легко получить постоянную ставку дисконтирования, требуемую методом цепи замещения, но невозможно в других. Если муниципальное правительство финансирует проекты с помощью облигаций с [общим обязательством](/generalobligationbond),. правительство может получить постоянную учетную ставку. Муниципальное правительство могло просто выпустить десятилетнюю облигацию и использовать полученные средства для одного десятилетнего проекта или двух последовательных пятилетних проектов. Если муниципальное правительство вместо этого выпускает [доходные облигации](/revenuebond),. оно должно финансировать проекты по мере их возникновения. В этом случае ставка дисконтирования может значительно измениться через пять лет. Уникальные возможности для финансирования, такие как [Build America Bonds](/build-america-bonds-babs),. также приходят и уходят.

## Альтернативы методу цепочки замещения

Метод замещающей цепи — не единственный способ оценить взаимоисключающие проекты с неравным сроком службы. Метод эквивалентного годового аннуитета ( [EAA](/equivalent-annual-annuity-approach) ) является альтернативным методом. Подход EAA заключается в оценке каждого проекта на основе прогнозируемого потока ренты (серии равных платежей). Для этого сначала вычисляется чистая приведенная стоимость каждого проекта, а затем преобразуется каждый в эквивалентный аннуитет. При таком подходе проект с наивысшим EAA считается более желательным.

Какой метод лучше подходит для принятия решений о капиталовложениях? Поскольку и цепочка замещения, и модели EEA основаны на расчетах чистой приведенной стоимости по сравнению с внутренней нормой доходности ( [IRR](/irr) ), они должны прийти к одним и тем же выводам. Отличаются только подходы.

## Особенности

- Метод цепи замещения представляет собой модель принятия решений по бюджету капиталовложений, которая сравнивает два или более взаимоисключающих предложения капиталовложений с неравным сроком службы.

- Метод замещающей цепочки включает многократное повторение более коротких проектов, пока они не достигнут срока жизни самого длинного проекта.

- Метод замещающей цепи требует повторяющихся проектов и постоянной ставки дисконтирования.
