# Интерфейс прикладного программирования (API)
Интерфейс прикладного программирования или API — это фрагмент кода, который позволяет двум приложениям обмениваться информацией.

Хорошим примером API является сайт-агрегатор бронирования отелей. Сайт-агрегатор использует API для запроса информации от различных гостиничных сетей в ответ на запрос клиента.

Например, клиент запрашивает информацию о доступных номерах, отвечающих определенным критериям — дата начала и окончания проживания, географический район, стоимость. Эта информация отправляется в системы отеля с помощью API, который переводит запрос в формат, понятный серверам отеля. Затем эти системы отвечают через API, указывая доступные им комнаты, которые соответствуют этим критериям.

API позволяет быстро и легко обмениваться информацией, несмотря на то, что сайт-агрегатор и гостиничная система построены с использованием разных технологий и разными компаниями.

В этом случае отель является поставщиком API. Он создал API, позволяющий другим приложениям получать доступ к информации. Отель делает это, потому что хочет, чтобы пользователи могли легко узнать о доступных номерах. Сайт-агрегатор является потребителем API. Он использует API, потому что было бы либо невозможно, либо слишком много работы для получения необходимых данных без него.

## Для чего можно использовать API?

API-интерфейсы помогают разработчикам использовать существующие функции и данные, а не использовать обходной путь или создавать его самостоятельно. Например, используя API Карт Google для отображения местоположения магазина или ресторана, разработчик избегает необходимости рисовать или программировать функции карты с нуля. Это значительно экономит время и деньги.

API-интерфейсы существуют везде, где необходимо взаимодействие различных частей программного обеспечения, и, поскольку предоставление данных полезно для бизнеса, многие из них можно использовать бесплатно.

Блокчейн, например, предоставляет бесплатные API, которые позволяют разработчикам получать доступ к обработке биткойн-платежей, услугам кошелька, данным транзакций и рыночным данным для использования на своих веб-сайтах и в приложениях.

Криптовалютные биржи также предоставляют API. Трейдеры могут использовать эти API для предоставления рыночных данных торговым ботам, позволяя им совершать сделки (в соответствии с заданными инструкциями) от имени трейдеров. Эта форма торговли известна как алгоритмическая торговля (или бот-трейдинг).

## Особенности

- Для финансовых рынков API-интерфейсы взаимодействуют с торговыми алгоритмами или моделями и биржевой и/или брокерской платформой.

- Все больше брокеров делают свои платформы доступными через API.

- Интерфейс прикладного программирования (API) устанавливает онлайн-соединение между поставщиком данных и конечным пользователем.

- API необходим для реализации автоматической торговой стратегии.
