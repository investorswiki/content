# Межпланетная файловая система (IPFS)
Межпланетная файловая система (IPFS) — это проект с открытым исходным кодом, который создает одноранговую файловую систему. Проект IPFS — это амбициозный проект, который потенциально может изменить то, как мы используем Интернет. Его конечная цель — соединить все устройства с одной и той же файловой системой, но не так, как это делает сегодня Интернет. Чтобы понять основы IPFS, полезно сравнить его с HTTP.

По существу, современная Всемирная паутина основана на протоколах HTTP и HTTPS. Короче говоря, это прикладные протоколы, которые обеспечивают передачу данных и доступность по всему миру.

HTTP работает как протокол запроса-ответа, который соединяет пользователей (клиентов) с серверами в зависимости от их местоположения. Например, если Алиса зайдет на веб-сайт, ее веб-браузер запросит содержимое на хост-сервер (например, Amazon Web Services). Если все пойдет хорошо, сервер AWS вернет ей веб-страницы.

Однако содержимое не является постоянным, поскольку оно поддерживается централизованным сервером. А поскольку на AWS размещается большое количество веб-сайтов, если их серверы по какой-либо причине выходят из строя, вместе с этим падает и большой процент Интернета.

Напротив, IPFS позволяет создавать постоянную и распределенную сеть, где можно хранить и совместно использовать все виды цифровых данных. В то время как связь HTTP зависит от сервера (с определенным местоположением) для предоставления контента, связь IPFS фокусируется на самом контенте.

С IPFS Алиса больше не будет запрашивать контент напрямую на сервере AWS. Вместо этого она спрашивала распределенную сеть: «Кто может предоставить мне этот контент?» и самые близкие сверстники быстро откликнутся.

В зависимости от реализации IPFS может иметь много преимуществ по сравнению с HTTP, таких как устойчивость к цензуре, целостность данных, более низкие эксплуатационные расходы, лучшая производительность и безопасность.

Некоторые ограничения связаны с низкими стимулами к участию в сети, что приводит к небольшому количеству пиров. Ограниченное внедрение затрудняет постоянную доступность файлов. Если определенные данные размещаются только на нескольких узлах и все отключаются, они станут недоступными.
