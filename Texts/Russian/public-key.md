# Открытый ключ
## Что такое открытый ключ?

Открытый ключ — это криптографический код, который позволяет пользователям получать [криптовалюту](/cryptocurrency) на свои счета. Открытый ключ и закрытый ключ — это инструменты, необходимые для обеспечения безопасности криптоэкономики.

## Как работает открытый ключ

Когда пользователь инициирует свою первую транзакцию с [биткойнами](/bitcoin) или [альткойнами](/altcoin),. создается уникальная пара открытого ключа и [закрытого ключа](/private-key). Каждый из ключей состоит из длинной строки буквенно-цифровых символов, которые помогают обеспечить безопасность активов пользователя в цифровой экосистеме.

Закрытый ключ известен только пользователю и служит цифровым идентификатором пользователя. Закрытый ключ разрешает пользователю тратить, снимать, переводить или выполнять любую другую транзакцию со своей учетной записи. Сложный алгоритм применяется к закрытому ключу для генерации открытого ключа, и оба ключа хранятся в [цифровом](/digital-wallet) [кошельке](/digital-wallet).

Когда пользователь инициирует транзакцию для отправки, скажем, биткойнов другому лицу, транзакция должна быть передана в сеть, где [распределенные узлы](/distributed-applications-apps) подтверждают действительность транзакции перед ее завершением и записью в [блокчейне](/blockchain).

Перед трансляцией транзакция подписывается цифровой подписью с использованием закрытого ключа. Подпись подтверждает право собственности на закрытый ключ, хотя никому не раскрывает детали закрытого ключа. Поскольку открытый ключ создается из закрытого ключа, открытый ключ пользователя используется для доказательства того, что цифровая подпись была получена из его закрытого ключа. Как только транзакция подтверждена как действительная, средства отправляются на публичный адрес получателя.

Публичный адрес — это хешированная версия открытого ключа. Поскольку открытый ключ состоит из очень длинной цепочки чисел, он сжимается и укорачивается для формирования публичного адреса. По сути, закрытый ключ генерирует открытый ключ, который, в свою очередь, генерирует публичный адрес.

Когда два человека заключают соглашение, в котором один отправляет другому токены или монеты, они раскрывают друг другу свои публичные адреса. Публичный адрес подобен номеру банковского счета. Отправителю нужен номер, чтобы иметь возможность отправить средства получателю, который затем сможет потратить или вывести их с помощью своего закрытого ключа. Получатель также может проверить партию монет отправителя, используя публичный адрес отправителя, который будет отображаться на его или ее экране.

## Особые соображения

Хотя открытый ключ и адрес вырабатываются из закрытого ключа, обратный случай почти невозможен.

Криптовалютная сеть остается безопасной благодаря использованию сложных математических функций, гарантирующих невозможность получения закрытого ключа из открытого ключа, тем более что открытый ключ и его хэш-версия видны всем в сети.

Поскольку невозможно восстановить закрытый ключ из открытого ключа или адреса, если пользователь потеряет свой закрытый ключ, любой биткойн или альткойн, расположенный по его публичному адресу, будет навсегда недоступен. С другой стороны, пользователь, потерявший свой открытый ключ, может воссоздать его с помощью закрытого ключа.

## Особенности

- Пользователям выдается закрытый ключ и открытый ключ при первой инициации транзакции.

- Открытый ключ используется для проверки цифровой подписи, подтверждающей право собственности на закрытый ключ.

- Закрытый ключ доступен только его пользователю и разрешает пользователю совершать транзакции со своей учетной записи.

- Открытый ключ — это криптографический код, используемый для облегчения транзакций между сторонами, позволяющий пользователям получать криптовалюты на свои счета.
