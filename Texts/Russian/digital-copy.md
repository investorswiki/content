# Цифровая копия
## Что такое цифровая копия?

Цифровая копия применительно к криптовалюте означает дублирующую запись каждой подтвержденной транзакции, имевшей место в одноранговой сети, такой как сеть [биткойнов](/bitcoin).

Цифровая копия — это функция безопасности протокола биткойнов, предназначенная для решения проблемы двойного расходования средств.

## Как работает цифровая копия

Подъем [криптовалют](/cryptocurrency) начался в 2009 году с появлением [биткойна](/bitcoin). Одним из катализаторов создания биткойна было стремление к цифровой валюте, которая не могла бы контролироваться какой-либо центральной властью и не требовала бы доверенной третьей стороны для гарантии транзакций.

В отличие от банковских транзакций, здесь нет центральной записи или базы данных транзакций биткойнов. Вместо этого биткойн работает через децентрализованную сеть независимых компьютеров, каждый из которых ведет отдельный учет всех подтвержденных транзакций. Этот тип распределенного реестра известен как [блокчейн](/blockchain).

## Проблема двойной траты

Операции в цифровой валюте с использованием [децентрализованной системы](/decentralizedmarket) привели к проблеме, известной как двойная трата. Двойная трата происходит, когда кто-то пытается отправить одну и ту же монету на два разных адреса.

В традиционной валюте двойные траты предотвращаются такими учреждениями, как банки, расчетные палаты и системы онлайн-платежей, которые проверяют остатки на счетах и историю транзакций и обнаруживают [овердрафты](/overdraft). Старые системы цифровой валюты, такие как [eCash](/ecash),. не имели удовлетворительного способа предотвращения двойных расходов и не имели успеха.

Чтобы решить эту проблему, изобретатель биткойна создал процесс, при котором каждая законная транзакция независимо распределяется и проверяется несколькими майнерами, распределенными по сети.

## Распределенный реестр и несколько цифровых копий

Каждая биткойн-транзакция передается майнеру, который собирает сотни транзакций в блок. Когда новый блок завершен, майнер передает блок сотням других биткойн-узлов, каждый из которых сравнивает новые транзакции со своей собственной цифровой копией блокчейна. Если какой-либо узел обнаруживает двойную трату, новый блок отклоняется. В противном случае узлы будут передавать новый блок другим узлам и майнерам.

Эта система эффективно предотвращает двойные траты, поощряя честное поведение и наказывая недобросовестных участников. Поскольку майнеры получают [вознаграждение за блок](/block-reward),. они финансово заинтересованы в принятии только законных транзакций. Если майнер не отклонит двойную трату, его блок не будет использоваться другими узлами.

## Другие проблемы с двойной тратой

Хотя цифровые копии обычно защищают сеть биткойн, бывают редкие обстоятельства, при которых двойная трата может остаться незамеченной. Это связано с правилом самой длинной цепочки: когда есть две конкурирующие версии блокчейна, самая длинная цепочка считается наиболее авторитетной.

Наиболее известный способ использования этого свойства — атака 51%. Если злоумышленник контролирует большую часть [вычислительной мощности сети](/hash),. он может тайно создать отдельную более длинную версию блокчейна с другими транзакциями. Когда публикуется вторая версия блокчейна, она фактически отменяет любые транзакции, которые были выполнены в более короткой цепочке.

Также возможна непреднамеренная отмена транзакций из-за вероятностного характера блокчейна. Если два майнера по отдельности одновременно обнаружат новые блоки, обе версии останутся в сети до тех пор, пока не будет добыт следующий блок. Когда это произойдет, одна версия будет принята сетью, а другая будет отклонена как [потерянный блок](/orphan-block-cryptocurrency).

По этой причине биткойн-транзакции не следует считать действительно «окончательными», пока они не станут частью цепочки с еще шестью блоками, добытыми после транзакции. Это связано с тем, что крайне маловероятно, что шесть блоков будут реверсированы. По крайней мере, в одном случае пользователь биткойнов смог повторно использовать биткойны, которые уже были потрачены в потерянном блоке.

## Особенности

- Хотя эта система предотвращает двойную трату, существуют некоторые обстоятельства, при которых биткойны могут быть потрачены дважды, например, при атаке 51%.

- [Биткойн](/bitcoin) -транзакции не хранятся в центральной базе данных. Вместо этого тысячи цифровых копий [блокчейна](/blockchain) хранятся в биткойн-узлах, которыми управляют пользователи по всему миру.

- Поскольку нет единого органа по транзакциям в блокчейне, злоумышленник не может изменить историю транзакций, не имея доступа к каждой цифровой копии.
