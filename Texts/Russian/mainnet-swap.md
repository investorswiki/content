# Обмен основной сети
По сути, своп основной сети состоит из переключения с одной сети блокчейна на другую. В большинстве случаев своп происходит, когда проект криптовалюты мигрирует со сторонней платформы (например, Ethereum) на собственную собственную сеть блокчейнов. На этом этапе их токены криптовалюты постепенно заменяются вновь выпущенными монетами, и вся активность блокчейна перемещается в новую цепочку.

Возьмем в качестве примера [БНБ .](/binance-coin-bnb) После запуска основной сети [Binance](/mainnet) Chain пользователям было предложено перейти с блокчейна Ethereum на Binance Chain. Поэтому держатели токенов [ERC-20](/erc-20) BNB начали заменять свои токены недавно выпущенной монетой BEP2 BNB (собственная монета Binance Chain). Обмен основной сети осуществлялся в соотношении 1:1, так что 1 BNB ERC-20 имел ту же стоимость, что и 1 BNB BEP2. После обмена все оставшиеся токены ERC-20 BNB были сожжены, поэтому теперь можно использовать только BNB новой цепочки.

Таким образом, обмен основной сети происходит, когда проект блокчейна заменяет ранее выпущенные токены своей новой криптовалютой, которая обычно работает в их собственной сети блокчейна. Этот процесс также может называться «миграцией токена». Обычно своп основной сети начинается сразу после запуска основной сети.

Следует отметить, что миграция токенов не всегда связана исключительно с запуском нового блокчейна. Это также может иметь место в ситуациях, когда проекты просто переходят с одного протокола на другой. Например, Storj выполнила замену основной сети, перенеся свои токены из протокола на основе биткойнов в сеть Ethereum, чтобы избежать проблем с масштабируемостью.

Каждый своп основной сети имеет свои особенности с точки зрения выполнения. Некоторые проекты назначают периоды миграции с предопределенным сроком для пользователей, чтобы обменять свои старые токены на новые. Если они этого не сделают, они могут потерять доступ к своим средствам, потому что старые токены обычно уничтожаются или замораживаются.

Напротив, своп основной сети, выполненный Binance, дал пользователям свободу оставлять свои токены BNB в своей учетной записи Binance, поэтому обмен мог быть автоматически выполнен командой Binance. На самом деле своп производился через заявки на вывод средств. Другими словами, сохраняя свои ERC-20 BNB в своей учетной записи Binance, пользователи могли позже вывести недавно выпущенные BEP2 BNB в равных количествах, не беспокоясь о ручном обмене своих токенов.
