# 도서관
컴퓨터 프로그래밍에서 라이브러리라는 용어는 실행 파일, 문서, 메시지 템플릿 및 작성된 코드를 포함할 수 있는 잘 문서화된 안정적인 리소스 모음을 나타냅니다.

이 용어는 사용 시 특정 작업을 수행하거나 특정 출력을 생성하는 미리 작성된 모듈 모음을 나타낼 수도 있습니다. 라이브러리의 모듈은 코드, 클래스 정의, 프로시저, 스크립트 및 구성 데이터의 형태를 취할 수 있습니다. 이는 개발자가 각 모듈에 대한 전체 코드를 처음부터 다시 작성할 필요 없이 라이브러리의 모듈을 호출할 수 있음을 의미합니다. 즉, 새 모듈을 자체적으로 생성하는 대신 다른 라이브러리에서 제공하는 동작을 재사용할 수 있습니다.

라이브러리는 전문화되지 않은 사용(예: 시간 말하기, 기본 수학 연산)을 위해 일반화되거나 특정 사용 사례(예: 사운드 파일 압축 디코딩)에 대해 매우 구체적일 수 있습니다.

따라서 코드 라이브러리를 사용하면 개발자와 컴퓨터 과학자에게 많은 이점이 있습니다. 그 중 일부는 시간과 자원을 보다 효율적으로 사용하여 개발자가 소프트웨어를 더 쉽게 개발하고 실행할 수 있도록 하는 것입니다.

코드 라이브러리 사용의 한 가지 단점은 개발자가 사용의 입력과 출력만 볼 수 있는 이른바 블랙박스 효과입니다.
