# 행성간 파일 시스템(IPFS)
IPFS(InterPlanetary File System)는 P2P 파일 시스템을 구축하는 오픈 소스 프로젝트입니다. IPFS 프로젝트는 우리가 인터넷을 사용하는 방식을 잠재적으로 바꿀 수 있는 야심찬 프로젝트입니다. 최종 목표는 동일한 파일 시스템을 가진 모든 장치를 연결하는 것이지만 오늘날의 웹과 같은 방식은 아닙니다. IPFS의 기본을 이해하려면 HTTP와 비교하는 것이 도움이 됩니다.

기본적으로 현재 World Wide Web은 HTTP 및 HTTPS 프로토콜을 기반으로 합니다. 요컨대, 이들은 전세계 데이터 통신 및 액세스를 가능하게 하는 애플리케이션 프로토콜입니다.

HTTP는 위치에 따라 사용자(클라이언트)를 서버에 연결하는 요청-응답 프로토콜로 작동합니다. 예를 들어 Alice가 웹사이트에 가면 그녀의 웹 브라우저는 콘텐츠를 호스팅 서버(예: Amazon Web Services)에 요청할 것입니다. 모든 것이 잘되면 AWS 서버는 웹 페이지를 그녀에게 반환합니다.

그러나 콘텐츠는 중앙 집중식 서버에 의해 유지 관리되기 때문에 영구적이지 않습니다. 그리고 AWS는 많은 수의 웹 사이트를 호스팅하고 있기 때문에 어떤 이유로든 서버가 다운되면 인터넷의 많은 부분도 함께 다운됩니다.

대조적으로, IPFS는 모든 종류의 디지털 데이터를 저장하고 공유할 수 있는 영구적이고 분산된 웹을 만들 수 있습니다. HTTP 통신이 콘텐츠를 제공하기 위해 서버(특정 위치가 있는)에 의존하는 반면, IPFS 통신은 콘텐츠 자체에 초점을 맞춥니다.

IPFS를 사용하면 Alice는 더 이상 AWS 서버에 직접 콘텐츠를 요청하지 않습니다. 대신 그녀는 분산 네트워크에 "누가 나에게 이 콘텐츠를 제공할 수 있습니까?"라고 묻곤 했습니다. 가장 가까운 동료가 신속하게 응답할 것입니다.

구현에 따라 IPFS는 검열 저항, 데이터 무결성, 낮은 운영 비용, 더 나은 성능 및 보안과 같은 HTTP와 비교할 때 많은 이점을 제공할 수 있습니다.

일부 제한 사항은 네트워크 참여에 대한 낮은 인센티브와 관련이 있으며, 이는 소수의 동료로 이어집니다. 제한된 채택으로 인해 파일을 영구적으로 사용하기가 더 어려워집니다. 특정 데이터가 소수의 노드에서만 호스팅되고 모두 오프라인 상태가 되면 액세스할 수 없게 됩니다.
