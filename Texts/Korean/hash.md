# 해시시
[암호화](/cryptography) 에서 해시라는 단어는 해시 함수를 통해 데이터 조각이 제출(매핑)된 후 해시 함수에 의해 생성된 출력을 나타냅니다. 단순한 해시 외에 해시 함수에 의해 생성된 출력은 해시 값, 해시 코드 또는 다이제스트라고도 할 수 있습니다.

해시가 무엇인지 더 잘 이해하려면 해시 함수가 무엇이고 어떻게 작동하는지 논의할 가치가 있습니다.

해시 함수는 모든 크기의 입력 값을 고정 크기의 출력(해시)으로 변환하는 수학적 [알고리즘 입니다.](/algorithm) 대부분의 경우 출력은 16진수로 구성됩니다. 이는 해시가 종종 숫자(0~9)와 문자(a~f)의 조합으로 표시됨을 의미합니다.

예를 들어 "Binance"라는 단어를 입력 값으로 사용하고 SHA-256 해시 함수를 통해 매핑하면 반환되는 출력 값(또는 해시)은 다음과 같습니다.

f1624fcc63b615ac0e95daf9ab78434ec2e8ffe402144dc631b055f711225191

이 작업을 몇 번 수행하든 상관없이 출력은 항상 동일합니다(입력이 변경되지 않는 한).

반면에 입력에 대한 사소한 변경으로 인해 해시 함수는 완전히 다른 해시를 출력으로 반환합니다. 예를 들어 "Binance" 대신 "binance"라는 단어를 제출하면 결과적으로 다음 해시를 갖게 됩니다.

59bba357145ca539dcd1ac957abc1ec5833319ddcae7f5e8b5da0c36624784b2

해시는 정보가 무엇인지 밝히지 않고 특정 정보의 유효성을 확인하는 데 유용합니다. 실제로 해시 함수는 다양한 시나리오에 적용될 수 있습니다. 몇 가지 사용 사례에는 데이터베이스 조회, 대용량 파일 분석 및 데이터 관리가 포함됩니다.

암호화 기술과 결합하면 소위 암호화 해시 기능이 있습니다. 이들은 정보 보안에 광범위하게 사용되며 대부분의 블록체인 네트워크에서 필수적인 부분입니다.

예를 들어, 비트코인 블록체인에는 해싱과 관련된 많은 작업이 있으며 이는 채굴 과정에서 중요합니다.

## 하이라이트

- 해시는 블록체인 계산을 해결하기 위해 필요한 암호화된 요구 사항을 충족하는 기능입니다.

- 해시는 논스나 솔루션과 같이 블록체인 네트워크의 중추입니다.

- 블록 헤더에 있는 정보를 기반으로 해시가 개발됩니다.

- 누군가가 블록체인을 해독하려고 시도하는 경우 해시의 길이를 추측하는 것이 거의 불가능하기 때문에 해시는 고정된 길이입니다.

- 동일한 데이터는 항상 동일한 해시 값을 생성합니다.

## 자주하는 질문

### 해시 값은 어떻게 계산되나요?

해시 함수는 임의 길이의 데이터를 고정 길이(예: 256자)의 데이터로 변환하는 복잡한 수학적 알고리즘을 사용합니다. 원본 데이터의 아무 곳에서나 한 비트만 변경하면 전체 해시 값이 변경되므로 디지털 파일 및 기타 데이터의 정확도를 확인하는 데 유용합니다.

### 블록체인에서 해시는 무엇에 사용됩니까?

해시는 블록체인 시스템의 여러 부분에서 사용됩니다. 첫째, 각 블록에는 이전 블록의 [블록 헤더 해시가 포함되어](/block-header-cryptocurrency) 있으므로 새 블록이 추가될 때 아무 것도 변조되지 않도록 합니다. 또한 [작업 증명](/proof-work) (PoW) 을 사용한 암호화폐 채굴 은 일련의 선행 0을 포함하는 특정 해시 값에 도달하기 위해 무작위로 생성된 숫자의 해시를 활용합니다. 이 임의의 기능은 리소스를 많이 사용하므로 악의적인 행위자가 네트워크를 추월하기 어렵습니다.

### 해시 함수란 무엇입니까?

해시 함수는 주어진 데이터 세트를 "해시 값"이라고도 하는 고정 크기의 비트 문자열로 변환하거나 "매핑"하는 수학 함수입니다.
