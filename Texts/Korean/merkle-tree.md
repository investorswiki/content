# 머클 나무
Merkle 트리는 대량의 데이터를 조직화하고 구조화하여 처리하기 쉽게 만드는 방법입니다. 암호화폐와 [블록체인 의 경우](/blockchain) 머클 트리는 리소스를 덜 요구하는 방식으로 트랜잭션 데이터를 구조화하는 데 사용됩니다.

Merkle 트리 구조에서 암호화폐 거래가 이루어지면 해시된 후 이에 상응하는 해시 값이 부여됩니다. Merkle 트리에서 각 트랜잭션이 해시된 후 생성된 해시 값은 다른 해시 값과 쌍을 이루고 다시 해시됩니다. 예를 들어 해시 값 'AB'와 'AC'를 결합하여 'ABC'를 만듭니다.

이러한 해시 값을 페어링하는 과정은 최종 해시 값이 생성될 때까지 반복됩니다. 최종 해시 값인 Merkle 루트는 포함된 모든 트랜잭션의 요약을 제공합니다. 그런 다음 Merkle 루트 요약이 블록 헤더에 삽입됩니다.

####데이터 보안

[블록](/block) 의 트랜잭션에 대한 액세스 기록을 쉽게 제공합니다 . 따라서 블록의 데이터가 변경되었거나 변조되었는지 여부를 확인하는 것은 매우 간단합니다. 이는 Merkle 트리의 트랜잭션(또는 기타 관련 데이터)에 대한 변경이 완전히 다른 해당 Merkle 루트로 이어지기 때문에 사실입니다.

#### 자원의 효율적인 사용

암호화폐가 Merkle 트리를 사용하지 않았다면 모든 검증 요청에는 네트워크를 통해 전송되는 엄청난 양의 정보가 포함될 것입니다. Merkle 트리에서 트랜잭션 데이터를 구조화하는 것은 리소스를 훨씬 더 효율적으로 사용합니다. 해시된 트랜잭션 데이터가 Merkle 루트에서 검증될 수 있기 때문에 트랜잭션을 검증하는 데 원장의 완전한 사본이 필요하지 않습니다. 따라서 노드를 통해 전송되는 정보가 훨씬 줄어들고 전체 데이터 무결성을 분석하기 위한 컴퓨팅 성능이 줄어듭니다.

즉, 머클 트리 구조를 통해 사용자는 전체 블록체인을 다운로드하는 과정을 거치지 않고도 개별 트랜잭션이 블록에 포함되었는지 확인할 수 있습니다. 이 기술은 암호화폐가 거래 데이터를 구성하고 효율적으로 작동하는 데 중요한 도구입니다. Merkle 트리가 없으면 리소스에 대한 수요가 증가 하면 네트워크에 참여하는 [노드](/node) 가 줄어들 수 있습니다.
