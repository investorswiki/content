# 데이터 마이그레이션
## 데이터 마이그레이션이란 무엇입니까?

데이터 마이그레이션은 컴퓨터, 시스템 또는 형식 간에 저장된 디지털 정보를 이동하는 프로세스입니다. 데이터 마이그레이션은 서버 교체 또는 유지 관리, [데이터 센터 변경](/data-warehousing),. 데이터 통합 프로젝트 및 시스템 업그레이드 를 비롯한 여러 가지 이유로 발생 합니다. 회사의 많은 기업 지식과 [비즈니스](/business-intelligence-bi) [인텔리전스](/business-intelligence-bi) 가 데이터에 포함되어 있으므로 모든 데이터 마이그레이션 프로젝트는 위험을 최소화하기 위해 신중하게 수행되어야 합니다.

## 데이터 마이그레이션 이해

제대로 수행되지 않을 경우 [비즈니스 연속성](/business-continuity-planning) 에 심각한 위험을 나타냅니다 . 물론 데이터 손실은 최악의 시나리오이지만 기업은 가동 중지 시간, 호환성 문제 및 전반적인 시스템 성능 문제도 해결해야 합니다. 데이터 마이그레이션은 많은 양의 데이터, 다양한 형식의 데이터, 기업 내 다양한 데이터 습관으로 인해 더욱 어려워집니다.

[백업](/back-up),. 이동 순서 및 동시 데이터 환경 을 우선시하는 상세한 데이터 마이그레이션 정책을 만듭니다 . 회사에서 새 환경이 준비되는 동안 사전 마이그레이션 환경을 실행할 수 없는 경우 데이터 마이그레이션을 허용하기 위해 현재 애플리케이션의 비즈니스 운영이 일시 중단되므로 상당한 다운타임이 발생합니다. 이러한 유형의 중지, 전송 및 시작 데이터 마이그레이션은 새 플랫폼으로 이동하거나 물리적 스토리지에 대한 엄격한 제한이 있고 기존 스토리지 기술에 대한 스왑 또는 수정이 필요한 경우 필요할 수 있습니다.

## 제로 다운타임 데이터 마이그레이션

제로 다운타임 마이그레이션 모델은 두 개의 완전한 환경을 생성하고 실행할 수 있는 충분한 스토리지가 있어야 합니다. 회사 데이터의 전체 복사본을 새 환경으로 가져와 직원이 이전 환경에 머무르는 동안 테스트합니다. 버그는 새 시스템에서 해결되어 모든 응용 프로그램이 여전히 작동하고 모든 것이 있어야 할 위치에 있도록 합니다. 테스트가 완료된 후 새 복사본을 가져와서 모든 직원을 새 환경으로 전환합니다. 직원이 이전 데이터 시스템에서 파일을 가져오지만 해당 서버에 새 데이터를 쓸 수 없도록 이전 데이터 환경을 몇 달 동안 열어 두는 경우가 있습니다. 모든 데이터 마이그레이션에서 데이터 손실을 확인하기 위해 마이그레이션 후 [데이터 감사 가 수행됩니다.](/audittrail)

## 데이터 마이그레이션 개선

데이터 마이그레이션을 개선할 수 있는 한 가지는 마이그레이션 전에 데이터 관행을 정리하고 표준화하는 것입니다. 회사 데이터의 구성은 사람들의 다양한 파일링 습관을 반영하는 경우가 많습니다. 같은 역할을 맡은 두 사람이 완전히 다른 방식을 사용할 수도 있습니다. 예를 들어, 한 경우에는 공급업체별로 계약을 저장하고 다른 경우에는 [회계 연도](/fiscalyear) 및 월별로 계약을 저장합니다. 데이터 관행을 통합하는 것은 실제로 데이터를 마이그레이션하는 것보다 훨씬 더 큰 작업일 수 있지만 명확한 정책에 의해 백업되는 깨끗하고 일관성 있게 구성된 데이터는 앞으로 있을 많은 마이그레이션에 대해 회사의 데이터를 미래에 보장하는 데 도움이 됩니다.

## 하이라이트

- 데이터 마이그레이션은 한 컴퓨터 시스템에 저장된 정보를 다른 컴퓨터 시스템으로 전환하는 것을 말합니다.

- 이 프로세스는 기술 업그레이드, 물리적 위치 간 이동 또는 일상적인 유지 관리 작업으로 인해 발생할 수 있습니다.

- 오늘날 기업은 일상적인 비즈니스를 수행하기 위해 데이터에 크게 의존하기 때문에 중단, 데이터 손실 또는 보안 침해가 발생하지 않도록 데이터 마이그레이션을 신중하고 안전하게 수행해야 합니다.
