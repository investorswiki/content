# 스냅 사진
전통적으로 스냅샷이라는 용어는 특정 시점에 컴퓨터 시스템이나 저장 장치의 상태를 기록하는 기능을 나타냅니다. 암호화폐에서 스냅샷은 종종 특정 [블록 높이 에](/block-height) [블록체인](/blockchain) 상태를 기록하는 행위를 설명합니다 . 이 경우 스냅샷은 모든 기존 주소와 관련 데이터(거래, 수수료, 잔액, 메타데이터 등)를 포함하는 전체 블록체인 원장의 내용을 기록합니다.

스냅샷은 일반적으로 각 라운드가 발생하기 전에 에어드롭 이벤트 중에 사용됩니다. 에어드롭 시 각 블록체인 주소의 잔고에 따라 토큰이 분배됩니다. 이 경우 특정 시점(즉, 블록 높이)에서 각 토큰 보유자의 잔액을 기록하기 위해 스냅샷이 찍힙니다. 대부분의 경우 사용자는 스냅샷이 생성된 후 해당 배포 라운드에 참여할 수 있는 자격을 손상시키지 않고 자금을 이동할 수 있습니다.

스냅샷은 블록체인 하드포크 중에도 중요합니다. 스냅샷은 새로운 체인을 생성하기 전에 메인 체인이 기록될 블록 높이를 표시하기 때문입니다. 예를 들어, 비트코인 캐시 하드 포크가 발생했을 때(2017년 8월 1일) 블록 478,558에 비트코인이 있는 모든 블록체인 주소의 잔액이 비트코인 캐시 블록체인에 복사되었습니다. 그 이유는 두 블록체인이 포크 이전에 동일한 기록 데이터를 공유하기 때문입니다. 분할이 완료되는 즉시 각 블록체인 네트워크는 독립적으로 작동합니다.

는 Binance Launchpad에서 진행되는 [IEO(Initial Exchange Offerings)](/initial-exchange-offering) 에 참여할 의사가 있는 사용자의 BNB 잔액을 기록하는 것 입니다. 그러나 스냅샷은 각 프로젝트의 지침에 따라 특정 규칙을 따릅니다. 어떤 경우에는 스냅샷이 매일 임의의 시간에 생성되고 사용자의 잔액은 미리 정의된 기간 내에서 평균이 됩니다.
