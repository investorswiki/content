# 암호화
## 암호화란 무엇입니까?

암호화는 한 컴퓨터 또는 장치에서 다른 컴퓨터 또는 장치로 저장되거나 전송되는 정보를 암호화 키로 사람이나 컴퓨터에서만 인코딩할 수 있는 방식으로 인코딩하는 데이터 보안 방법입니다. 주요 금융 기관에서 민감한 개인 정보를 보호하기 위해 가장 많이 사용하는 기술은 데이터 암호화 기술에 의존합니다.

매일 인터넷을 통해 주고받는 민감한 정보의 양으로 인해 더 많은 웹사이트와 온라인 상거래 서비스에서도 정교한 암호화 시스템을 채택하고 있습니다.

## 더 깊은 정의

데이터를 암호화하는 방법은 여러 가지가 있습니다.

- 개인 키 암호화는 네트워크의 각 컴퓨터가 다른 컴퓨터로 정보를 보내기 전에 정보를 암호화하는 데 사용되는 비밀 코드 또는 키를 가지고 있음을 의미합니다. 개인 키 암호화가 작동하려면 서로 "대화"하고 키가 설치된 컴퓨터를 알아야 합니다. 개인 키를 갖는 것은 두 컴퓨터 간에 전송된 메시지를 해독하기 위해 그들에게만 알려진 두 컴퓨터 간에 공유되는 비밀 코드와 유사합니다. 메시지가 다른 사람에게 표시되면 의미 없는 정보처럼 보입니다.

- 공개 키 암호화는 공개 키와 개인 키를 모두 사용합니다. 컴퓨터만 개인 키를 알고 있지만 보안 통신이 필요한 모든 컴퓨터에 공개 키를 제공합니다. 메시지를 받는 컴퓨터는 자신의 개인 키와 발신 컴퓨터에서 받은 공개 키를 사용하여 암호화된 메시지를 해독합니다.

민감한 데이터 전송에 의존하는 기업, 금융 산업에 종사하는 기업은 암호화 부족으로 인해 데이터 유출로 고통받고 있습니다. 손상된 평판을 처리해야 하는 것 외에도, 손상을 복구하는 데 상당한 재정 및 노동 비용이 필요할 수 있습니다.

민감한 금융 정보가 노트북이나 스마트폰과 같은 장치에 저장될 때 데이터가 안전하다고 생각하는 사용자는 종종 이를 간과합니다. 그러나 이러한 시스템은 여전히 해킹될 수 있으며 기밀 정보에 액세스할 수 있습니다. 해커나 도둑으로부터 데이터를 보호하려면 장치에 저장된 데이터를 암호화해야 합니다.

## 암호화 예

웹 기반 이메일 서비스 또는 기타 권한 기반 웹사이트에 로그인할 때마다 사용자 이름과 비밀번호를 입력하면 브라우저에서 암호화됩니다. 웹사이트를 호스팅하는 서버와 귀하의 컴퓨터 또는 모바일 장치는 암호화된 정보를 온라인으로 주고받는 데 사용할 프로토콜에 동의합니다.

암호화된 온라인 통신을 사용하면 다음과 같은 여러 이점을 얻을 수 있습니다.

- 당신이하는 일을 아는 사람들의 수를 줄입니다.

- 해커가 귀하의 로그인 세부 정보에 액세스하는 것을 더 어렵게 만듭니다.

- 브라우징 데이터를 가리는 것.

- 속아서 악성 웹사이트를 방문할 가능성을 줄입니다.

사람들이 인터넷에서 보내는 많은 시간은 자신의 정보를 다른 사람에게 보내는 것과 관련이 있습니다. 온라인에서 물건을 구매하거나 온라인 계정을 개설하기 위해 가입 양식을 작성할 때 많은 개인 정보를 입력하게 됩니다. 따라서 암호화가 필수적입니다. 자물쇠 아이콘이 표시되면 브라우저 내에서 온라인 통신이 안전하게 암호화되었는지 알 수 있습니다 URL 표시줄에 "https://"가 포함된 웹사이트 주소.

##하이라이트

- 암호화는 정확한 키나 암호를 가진 사람만 읽거나 해독할 수 있도록 수학적으로 인코딩하여 데이터를 보호하는 수단입니다.

- 디지털 암호화 프로세스는 승인된 사용자 외에는 원본 정보를 읽을 수 없도록 하는 알고리즘을 사용하여 데이터를 변환합니다.

- 암호화는 디지털로 연결된 세상에서 개인 정보, 메시지 및 금융 거래를 비공개로 안전하게 유지하는 데 매우 중요합니다.
