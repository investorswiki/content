# 신용카드 승인 키
## 신용카드 인증키는 무엇인가요?

신용 카드 승인 키는 [신용 카드](/creditcard) 거래를 처리하는 데 사용되는 코드입니다. 코드는 고객이나 판매자에게 표시되지 않지만 대신 판매자의 [POS(Point of Sale)](/point-of-sale) 시스템과 결제 처리 제공업체 간에 암호화된 형식으로 전송됩니다.

신용 카드 거래를 안전하게 처리하려면 "PKI" 또는 "공개 키 인프라"라는 데이터 암호화 방법을 사용해야 하기 때문에 "키"와 "코드"라는 용어는 종종 같은 의미로 사용됩니다.

지불 프로세서가 고객이 거래를 완료하기에 충분한 자금이 있음을 확인하면 신용 카드 승인 키가 POS 시스템으로 다시 전송되어 신용 카드 거래가 승인됩니다.

## 신용 카드 승인 키 작동 방식

신용 카드 승인 키는 신용 카드 거래 처리와 관련된 많은 구성 요소 중 하나일 뿐입니다. 이 프로세스의 실제 메커니즘은 상당히 복잡하지만 오늘날의 컴퓨터 시스템에서는 신용 카드 거래를 단 몇 초 만에 처리할 수 있습니다.

일반적으로 신용 카드 승인 키는 신용 카드 번호와 처리 중인 거래의 세부 정보 조합을 기반으로 생성됩니다. 이러한 세부 정보에는 청구되는 금액과 거래가 발생한 시간이 포함되는 경우가 많습니다. 이 정보가 POS 시스템에 의해 수집되면 해당 데이터는 암호화되어 결제 네트워크를 통해 전송 되어 고객의 신용 카드를 발급한 [은행 의 확인을 받습니다.](/bank)

이 정보를 받으면 발급 은행은 고객이 거래를 완료하기에 충분한 자금이 있는지 확인합니다. 동시에 해당 신용 카드가 [사기](/fraud),. 절도 또는 기타 그러한 목적으로 표시되었는지 확인합니다. 그러한 문제가 없고 필요한 자금이 사용 가능한 경우 발행 은행은 POS 단말기에 의해 해독되는 암호화된 인증 키를 반환합니다. 그러면 거래가 승인됩니다.

## 신용 카드 승인 키의 실제 예

Emma는 작은 소매점의 소유자입니다. 고객이 매장을 최대한 편리하게 이용할 수 있도록 Emma는 모든 주요 신용 카드 및 직불 카드를 사용할 수 있는 POS 단말기를 사용합니다. 그녀는 또한 고객이 온라인으로 구매하고 집으로 직접 배송받을 수 있는 온라인 상점을 운영하고 있습니다.

두 경우 모두 Emma는 신용 카드 승인 키를 사용하여 거래를 수락하거나 거부하는 지불 처리 시스템에 의존합니다. 고객이 실제 매장에서 신용카드로 결제하면 POS기기에서 카드번호와 해당 거래내역을 카드 소지자의 발급은행으로 전송하고 거래 승인 여부를 결정한다.

고객이 온라인으로 주문할 때 고객은 일반적으로 신용 카드 뒷면의 서명란 근처에 인쇄되어 있는 카드 확인 값 코드(CVC)도 제출해야 합니다. 이 코드는 신용 카드 인증 키를 획득하고 구매를 완료하기 위해 기타 세부 정보와 함께 카드 발급자에게 전송됩니다.

## 하이라이트

- 신용카드 인증키는 거래가 이루어지고 승인되었음을 증명하는 역할을 하여 사기 방지에 도움이 됩니다.

- 가맹점의 POS 시스템과 카드 소지자의 발급 은행간에 자동으로 전송됩니다.

- 신용카드 인증키는 신용카드 거래를 완료하기 위해 필요한 코드입니다.
