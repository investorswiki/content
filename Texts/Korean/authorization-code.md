# 인증 코드
## 인증 코드란 무엇입니까?

인증 코드는 사용자가 항목을 구매, 판매 또는 양도하거나 보안 보호 공간에 정보를 입력할 수 있는 권한을 부여하는 영숫자 암호입니다. 인증 코드는 일반적으로 개인의 신원을 확인하거나 거래를 승인하거나 보안 영역에 대한 액세스를 제공하는 일련의 문자, 숫자 또는 둘의 조합입니다.

는 고객의 신용 카드에 거래를 승인할 수 있는 충분한 신용이 있는지 확인하기 위해 [신용 카드 발급자](/creditcard) 가 판매자에게 보내는 것 입니다.

> 다른 플랫폼은 사용자에게 암호 생성을 요청할 때 특정 요구 사항을 부과할 수 있습니다. 예를 들어 대문자와 소문자 모두에 구두점을 사용하거나 최소 길이 임계값을 사용해야 할 수 있습니다.

>

## 인증 코드 이해하기

인증 코드는 사용자가 액세스할 수 있는 제한 사항이 있는 모든 거래 또는 항목에 사용됩니다. 예를 들어 신용 카드 승인 코드는 발급 은행에서 판매를 승인하는 공급업체까지의 5자리 또는 6자리 코드입니다. 사용하신 신용카드가 위조 또는 카드 한도 초과 시 신용카드사는 자동으로 판매를 거부합니다. 승인되면 승인 코드가 신용 카드 거래에 첨부됩니다. 이는 거래가 합법적임을 가맹점에 알리는 동시에 상품 반품 또는 구매 분쟁과 같은 후속 검사에서 거래를 식별하는 데 도움이 됩니다.

인증 코드는 디지털 방식으로 전송되며 [신용 카드 처리 를 가속화하는 데 사용됩니다](/batch-credit-card-processing). 공급업체가 모든 거래를 완료하기 위해 구두 승인 코드를 발급자에게 요청해야 한다면 상거래 속도가 크게 저하될 것입니다.

## 비용 승인 및 데이터 보안의 승인 코드

[기업 재무 통제](/internalcontrols) 에서도 역할을 합니다 . 직원은 구매 및 비용에 대해 다른 인증 코드를 제공받을 수 있습니다. 이를 통해 기업은 특정 영역에서 직원 수준까지 구매 및 지출을 추적할 수 있습니다.

이러한 인증 코드에는 특정 트랜잭션 임계값도 제공됩니다. 직원이 허용 한도를 초과하여 비용을 지출하려고 하는 경우 관리자, 감독자 또는 조직 계층의 상위 직원이 부여한 승인 코드가 필요합니다. 이러한 의미에서 승인 코드는 직원 사기 또는 자금 남용을 방지하는 데 사용할 수 있는 통합 제어 메커니즘입니다.

인증 코드는 정보 보안을 유지하기 위해 전문 작업장에서도 일반적으로 사용됩니다. 서버 또는 VPN에 대한 액세스는 고유한 사용자 ID에 연결된 인증 코드를 사용하여 관리하여 민감한 데이터베이스에 액세스할 수 있는 사람을 제어할 수 있습니다.

이러한 인증 코드는 직원 재직 기간 동안 영구적으로 사용될 수 있지만 암호 제어 새로 고침과 유사하게 주기적으로 새로 고쳐집니다. 단일 세션 기간 동안만 지속되는 일회성 인증 코드 또는 토큰도 있습니다.

##하이라이트

- **인증 코드는 사용자가 다양한 거래를 수행할 수 있도록 인증하는 영숫자 비밀번호입니다.**

- **기업 세계에서는 특정 거래 임계값이 포함된 구매 및 비용에 대해 직원에게 다른 인증 코드를 부여할 수 있습니다.**

- **인증 코드는 직원 재직 기간 동안 영구적으로 사용될 수 있지만 일상적인 갱신이 필요한 경우가 더 많습니다.**
