# 프로그레시브 웹 애플리케이션(PWA)
PWA(Progressive Web Application)라는 용어는 최신 웹 기술을 사용하여 만들어지고 기본 웹 표준을 따르는 특수한 종류의 응용 프로그램을 나타냅니다. 기본적으로 PWA는 기본 앱과 매우 유사하게 작동하는 웹 앱입니다. 하지만 웹 애플리케이션과 네이티브 애플리케이션의 차이점은 무엇입니까?

### 웹 앱과 기본 앱

한편으로 웹 앱은 웹 브라우저를 통해 쉽게 접근할 수 있기 때문에 (특히 신규 사용자가) 더 쉽게 접근할 수 있습니다. 웹 앱을 사용할 때 사용자는 컴퓨터나 모바일 장치에 응용 프로그램을 설치할 필요가 없습니다. 또한 웹 앱은 웹 서버에서 온라인으로 호스팅되므로 모든 종류의 장치 및 운영 체제에서 쉽게 업데이트하고 사용할 수 있습니다. 즉, 여러 앱 스토어나 사용자에게 직접 업데이트를 푸시할 필요가 없습니다.

반면에 기본 앱은 사용자가 기기에 로컬로 다운로드하여 설치해야 하는 앱입니다. 앱 스토어 또는 앱 마켓플레이스를 통해 정기적으로 다운로드하는 앱입니다. 일반적으로 기본 앱은 장치의 하드웨어 및 기본 기능(카메라, GPS 및 마이크 포함)과 더 잘 동기화되어 작동할 수 있기 때문에 웹 앱보다 속도와 성능이 더 뛰어납니다. 그러나 기본 앱은 개발 비용이 더 많이 들고 모든 기기와 항상 호환되는 것은 아닙니다(각 운영 체제에는 자체 버전의 앱이 필요함).

### 프로그레시브 웹 애플리케이션(PWA)

간단히 말해서, 프로그레시브 웹 애플리케이션(PWA)은 안정적이고 빠르며 매력적입니다. 기본 앱과 웹 앱의 장점을 모두 제공합니다. 따라서 웹 응용 프로그램임에도 불구하고 PWA는 모든 장치와 호환되며 실제로는 웹 브라우저 응용 프로그램보다 기본 앱처럼 보입니다. PWA는 가장 기본적인 브라우저 표준을 준수하는 모든 브라우저에서 작동할 수 있습니다.

일반적으로 PWA는 HTML, CSS 및 JavaScript와 같은 기술을 사용하여 향상된 속도와 훨씬 더 많은 기능으로 작동할 수 있습니다. 이름에서 알 수 있듯이 PWA는 전체 수명 동안 점진적으로 개선될 수 있습니다.

기존 웹 앱과 달리 PWA는 인터넷 연결에 엄격하게 의존하지 않습니다. 즉, 열악한 네트워크 조건이나 오프라인에서도 계속 실행할 수 있습니다. 또한 PWA는 백그라운드에서 실행될 수 있고 항상 최신 상태이며 간단한 링크로 쉽게 공유할 수 있습니다.

그 외에 PWA는 웹 트래픽에 계산됩니다. 즉, 기본 앱과 달리 전체 광고 및 검색 성능에 유익한 페이지 뷰 측면에서 웹 페이지의 분석을 향상시킬 수 있습니다.
