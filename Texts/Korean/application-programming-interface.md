# API(응용 프로그래밍 인터페이스)
애플리케이션 프로그래밍 인터페이스 또는 API는 두 애플리케이션이 정보를 공유할 수 있도록 하는 코드입니다.

API의 좋은 예는 호텔 예약을 위한 집계 사이트입니다. 애그리게이터 사이트는 API를 사용하여 고객의 요청에 대한 응답으로 다양한 호텔 체인의 정보를 요청합니다.

예를 들어, 고객은 특정 기준(숙박 시작 및 종료 날짜, 지리적 지역, 비용)을 충족하는 사용 가능한 객실에 대한 정보를 요청합니다. 이 정보는 API를 사용하여 호텔 시스템으로 전송되며, API는 요청을 호텔 서버가 이해할 수 있는 형식으로 변환합니다. 그런 다음 이러한 시스템은 API를 통해 이러한 기준과 일치하는 사용 가능한 회의실로 응답합니다.

API를 사용하면 애그리게이터 사이트와 호텔 시스템이 서로 다른 기술과 다른 회사에서 구축되고 있음에도 불구하고 정보를 빠르고 쉽게 교환할 수 있습니다.

이 시나리오에서 호텔은 API 공급자입니다. 다른 앱이 정보에 액세스할 수 있도록 API를 구축했습니다. 호텔은 사용자가 사용 가능한 객실을 쉽게 찾을 수 있도록 하기 위해 이렇게 합니다. 집계 사이트는 API의 소비자입니다. API가 없으면 필요한 데이터를 얻는 것이 불가능하거나 너무 많은 작업이 되기 때문에 API를 사용합니다.

## API는 무엇에 사용할 수 있나요?

API는 개발자가 해결 방법을 사용하거나 직접 구축하는 대신 기존 기능과 데이터를 사용할 수 있도록 도와줍니다. 예를 들어 Google Maps API를 사용하여 상점이나 레스토랑의 위치를 표시함으로써 개발자는 처음부터 지도 기능을 그리거나 코딩할 필요가 없습니다. 이것은 상당한 시간과 비용을 절약합니다.

API는 서로 다른 소프트웨어 조각이 상호 작용해야 하는 곳이면 어디에나 존재하며 데이터를 사용 가능하게 만드는 것이 비즈니스에 좋기 때문에 대부분이 무료입니다.

예를 들어 블록체인은 개발자가 웹사이트 및 애플리케이션에서 사용할 비트코인 결제 처리, 지갑 서비스, 거래 데이터 및 시장 데이터에 액세스할 수 있도록 하는 무료 API를 제공합니다.

암호화폐 거래소는 API도 제공합니다. 거래자는 이러한 API를 사용하여 거래 봇에 시장 데이터를 제공하여 거래 봇이 거래자를 대신하여 (사전 설정 지침에 따라) 거래할 수 있도록 합니다. 이러한 거래 형태를 알고리즘 거래(또는 봇 거래)라고 합니다.

##하이라이트

- 금융의 경우 API는 거래 알고리즘 또는 모델과 거래소 및/또는 브로커 플랫폼을 인터페이스합니다.

- 더 많은 브로커가 API를 통해 플랫폼을 제공하고 있습니다.

- API(응용 프로그래밍 인터페이스)는 데이터 공급자와 최종 사용자 간의 온라인 연결을 설정합니다.

- 자동화된 거래 전략을 구현하기 위해서는 API가 필수적입니다.
