# 거래 인증 번호(TAN)
## 거래 인증 번호(TAN)란 무엇입니까?

거래 인증 번호는 온라인 거래 처리에 사용되는 일회성 코드입니다. TAN(트랜잭션 인증 번호)은 계정에 안전하게 로그인하거나 트랜잭션을 수행하기 위한 암호 이상의 추가 보안 계층을 나타냅니다.

## 분류 거래 인증 번호(TAN)

TAN(트랜잭션 인증 번호)은 추가 보안을 제공합니다. 가맹점과 지불 카드 회사는 더 나은 보안이 사기가 발생할 가능성을 줄이기 때문에 거래 보안을 개선할 인센티브가 있습니다. 지불 카드 산업 보안 표준 위원회(Payment Card Industry Security Standards Council)와 같은 조직은 POI(Point-of-Interaction)에서 카드 정보를 암호화하고 나중에 트랜잭션을 해독하고 처리하기 위해 지불 카드 [취득자](/acquirer) 와 처리 회사가 따라야 하는 표준을 만듭니다.

거래 승인 번호는 금융 기관이 사기 가능성을 줄일 수 있는 한 가지 방법입니다. 이 번호는 일회용 번호이며 트랜잭션의 2단계 인증을 제공합니다. 인증의 첫 번째 수준은 계정에 액세스하기 위한 PIN( [개인 식별 번호](/personal-identification-number) ) 또는 암호를 포함할 수 있고, 두 번째 수준의 인증은 TAN일 수 있습니다.

금융 기관은 일반적으로 거래를 인증하는 데 사용할 수 있는 암호 또는 암호 목록을 제공하며 각 TAN은 한 번만 사용할 수 있습니다. TAN 목록을 제공하는 금융 기관은 각 TAN을 특정 사용자와 연결하는 데이터베이스를 유지 관리합니다.

TAN은 온라인 거래 확인에 가장 일반적으로 사용됩니다. 개인이나 기업이 거래를 시작하면 이메일, SMS 문자 메시지 또는 다른 방법을 통해 TAN과 함께 제공될 수 있습니다. 전달 방법은 일반적으로 은행이 특정 전화번호가 계좌에 연결되었음을 확인하는 문자 메시지를 보내는 것과 같은 이전 상호 작용을 통해 사전에 인증되었습니다. 트랜잭션이 수행될 때 사용자는 TAN 코드가 포함된 메시지를 받게 되며 웹 기반 양식의 필드에 해당 코드를 입력해야 합니다. 코드가 올바르게 일치하면 트랜잭션이 처리됩니다.

## 거래 인증 번호 및 이중 인증

미국인들이 더 많은 활동을 온라인으로 이동함에 따라 거래 인증 번호의 사용은 금융 기관에서 다른 많은 삶의 영역으로 확산되었습니다. 예를 들어 이메일 사용자는 [이중 인증](/twofactor-authentication-2fa) 을 사용하여 이메일 계정에 로그인하는 것이 좋습니다. 이 경우 거래 인증 번호와 함께 비밀번호를 입력해야 합니다. 이것은 일반적으로 사용자가 일회용 코드 목록에 보관하거나 문자 메시지, 이메일 또는 전화 통화를 통해 번호를 검색합니다.
