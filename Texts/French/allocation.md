# Allocation
L'allocation est une allocation de jetons ou de capitaux propres, qui peuvent être gagnés, achetés ou mis de côté pour un certain investisseur, équipe, groupe, organisation ou autre entité liée.

Pour une équipe de cryptographie en phase de livre blanc, l'équipe de direction ou la communauté peut travailler à décider quelles allocations de jetons doivent être divisées en différentes initiatives et départements, tels que le développement, le marketing, les coûts opérationnels, etc. Si une équipe a une fondation ou une autre entité qui contrôle les fonds, elle peut également décider de créer une allocation pour une trésorerie symbolique à utiliser comme spécifié par l'équipe ou la communauté.

Les investisseurs peuvent également recevoir des allocations lors de cycles d'investissements. Par exemple, une équipe peut vendre des allocations d'une «taille de ticket» définie, ou d'un montant maximum, aux premiers investisseurs lors d'un tour de vente privé. Dans ce cas, chacun de ces investisseurs individuels serait propriétaire d'une allocation sur le montant total offert lors de ce cycle de vente particulier. Une entité individuelle peut avoir le potentiel de détenir des allocations de plusieurs cycles de vente, ce qui signifie qu'elle pourrait éventuellement participer à différentes étapes d'une offre initiale de pièces (ICO) ou d'un événement de vente de jetons, avec une allocation prédéfinie pour chaque étape.

Les membres de l'équipe travaillant sur une monnaie, un protocole ou un projet spécifique peuvent également recevoir une partie d'une allocation d'équipe en récompense de leur travail. Par exemple, ces allocations pourraient être versées en une seule fois à une certaine date - comme la date d'un événement de génération de jetons (TGE) - ou étalées dans le temps, selon un calendrier prédéfini.

Dans certaines situations, les attributions sont réparties dans le temps dans le cadre d'une récompense globale ou sur une période d'acquisition ou une falaise.
