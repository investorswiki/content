# ID de transaction (TXID)
Un identifiant de transaction (TXID) ou hachage de transaction est une chaîne de caractères unique donnée à chaque transaction vérifiée et ajoutée à la blockchain. En d'autres termes, un TXID est un numéro d'identification qui étiquette chaque transaction sur la blockchain.

Lorsque vous effectuez une transaction et qu'elle est ajoutée à la blockchain, un identifiant de transaction lui est attribué en tant qu'identifiant unique. Selon la blockchain utilisée, vous pouvez ensuite la retrouver dans un explorateur de blocs en recherchant l'ID de transaction. Cela peut être utile pour vérifier les informations de base sur la transaction, telles que le montant envoyé, la date du transfert, l'adresse d'envoi/de réception et le nombre de confirmations de réseau que la transaction a reçues.

Par exemple, voici le TXID de la toute première transaction Bitcoin envoyée par Satoshi Nakamoto à Hal Finney :

**f4184fc596403b9d638783cf57adfe4c75c605f6356fbc91338530e9831e9e16**

Voici le TXID de la célèbre transaction [Bitcoin Pizza](/bitcoin-pizza) :

**cca7507897abc89628f450e8b1e0c6fca4ec3f7b34cccf55f3f531c659ff4d79**

Vous pouvez rechercher ces transactions vous-même en copiant les TXID dans un explorateur de blocs Bitcoin, tel que blockchain.com, [blockchair.com](/bitcoin) ou bien d'autres.

Lorsque vous retirez de la crypto-monnaie d'un échange centralisé (CEX) comme Binance, la plateforme vous montrera le TXID de votre transaction de retrait. Sur Binance, vous pouvez le trouver sur votre page Historique des transactions. Si vous cliquez sur un TXID pour un retrait donné, cela vous mènera à la transaction correspondante dans l'explorateur de blocs de cette blockchain.

Si vous envoyez des fonds à la mauvaise adresse (ou blockchain), le TXID dans votre historique de retrait est essentiel pour commencer à déterminer si vous pouvez récupérer vos fonds. Même dans ce cas, ce n'est pas une garantie et il est préférable de contacter l'assistance sur la plate-forme sur laquelle vous vous retirez.
