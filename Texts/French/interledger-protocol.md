# Protocole Interledger
## Qu'est-ce que le protocole Interledger ?

Le protocole Interledger est un protocole [blockchain](/blockchain) utilisé pour les paiements sur différents réseaux de paiement. Le protocole [open source](/open-source) connecte les registres de deux ou plusieurs banques différentes, supprimant ainsi les intermédiaires et les autorités centrales du système. Il promet de réduire les coûts et le temps requis pour traiter les transactions transfrontalières.

Interledger est utilisé par Ripple Labs pour connecter les systèmes bancaires à travers les frontières où le jeton [Ripple](/ripple-cryptocurrency) (XRP) fonctionne comme une couche de règlement standardisée entre les banques mondiales, ce qui rend Ripple quelque peu semblable à un service numérique [hawala](/hawala).

## Comprendre le protocole Interledger

Le processus actuel de virements transfrontaliers entre banques implique la navigation dans plusieurs protocoles de paiement utilisés par les registres numériques pour traiter les transactions. Les protocoles communiquent à l'aide de connecteurs, mais les outils et les normes utilisés dans de tels systèmes sont fragmentés.

Par exemple, l'ensemble actuel de connecteurs n'a pas de modes de communication normalisés. L'introduction de banques intermédiaires pour faciliter les transactions entre entités qui n'ont pas de relation directe les unes avec les autres complique davantage le processus. En effet, cela multiplie le nombre de sauts pour effectuer une transaction, ce qui la rend plus coûteuse et plus longue. Cela amplifie également les risques de sécurité car les paiements doivent transiter par plusieurs systèmes pour parvenir au destinataire final.

Le protocole Interledger utilise le concept de [séquestre cryptographique](/escrow) pour permettre le passage de fonds via des connecteurs ou des nœuds du réseau. Le livre blanc sur le protocole Interledger définit le dépôt fiduciaire cryptographique comme l'équivalent financier d'un protocole de validation en deux phases. Ce dernier protocole se compose de deux étapes :

1. La première étape consiste à définir un ensemble de conditions pour qu'une transaction avance ou abandonne.

1. La deuxième étape définit le traitement de la transaction une fois les conditions remplies.

Le séquestre cryptographique pour les grands livres est le verrouillage conditionnel des fonds entre deux parties. Les fonds ne sont débloqués qu'après que certaines conditions, y compris celles basées sur le temps pour que la transaction ait lieu, sont remplies. Dans le cas contraire, la transaction est annulée. À la fin d'une transaction de paiement réussie, l'expéditeur reçoit un reçu cryptographique du destinataire. Alternativement, les fonds entiercés sont retournés aux parties respectives.

## Mode atomique ou universel

Le protocole Interledger peut être implémenté en deux modes : le mode atomique et le mode universel. En mode atomique, les notaires sont intégrés au système. Il s'agit d'un groupe ad hoc utilisé pour vérifier et valider les transactions. En règle générale, les modes atomiques se déroulent entre des nœuds de connexion de confiance entre des banques ou des sociétés de services financiers qui peuvent être liées les unes aux autres .

Le mode universel ne nécessite pas de notaires et peut fonctionner entre des connecteurs non fiables. Il utilise la crypto-monnaie interne de Ripple, XRP, pour faciliter les transferts. Le transfert s'accompagne de contraintes de temps. Si elle n'a pas lieu dans un certain délai, la transaction est annulée .

## Points forts

- Interledger est une suite de protocoles open source permettant d'envoyer des paiements sur différents registres.

- Basé sur la technologie blockchain, l'objectif est de supprimer les intermédiaires pour créer un réseau de paiements interbancaires sécurisé, décentralisé et transfrontalier.

- Interledger n'est pas lié à une seule entreprise, blockchain ou devise, bien qu'il figure en bonne place avec la crypto-monnaie Ripple.
