# Migration de données
## Qu'est-ce que la migration de données ?

La migration des données est le processus de déplacement des informations numériques stockées entre les ordinateurs, les systèmes ou les formats. La migration des données se produit pour un certain nombre de raisons, notamment le remplacement ou la maintenance du serveur, un changement de [centres de données](/data-warehousing),. des projets de consolidation des données et des mises à niveau du système. Étant donné qu'une grande partie des connaissances d'entreprise et de l' [intelligence](/business-intelligence-bi) [commerciale d'une entreprise](/business-intelligence-bi) est contenue dans ses données, tout projet de migration de données doit être réalisé avec soin afin de minimiser les risques.

## Comprendre la migration des données

La migration des données représente un risque important pour [la continuité des activités](/business-continuity-planning) si elle n'est pas effectuée correctement. La perte de données est, bien sûr, le pire scénario, mais les entreprises doivent également faire face aux temps d'arrêt, aux problèmes de compatibilité et aux problèmes de performances globales du système. La migration des données est rendue plus difficile par de grandes quantités de données, divers formats de données et des habitudes de données divergentes au sein d'une entreprise.

Pour minimiser les risques de migration de données, les entreprises créent des politiques de migration de données détaillées qui donnent la priorité aux [sauvegardes](/back-up),. à l'ordre de déplacement et aux environnements de données simultanés lorsque cela est possible. S'il n'est pas possible pour une entreprise d'exécuter un environnement de pré-migration pendant la préparation du nouvel environnement, il y aura un temps d'arrêt important car les opérations commerciales sur les applications actuelles sont suspendues pour permettre la migration des données. Ce type d'arrêt, de transfert et de démarrage de la migration des données peut être nécessaire lors du passage à de nouvelles plates-formes ou lorsqu'il existe des limites strictes sur le stockage physique et que des échanges ou des correctifs sont nécessaires sur la technologie de stockage existante.

## Migration de données sans temps d'arrêt

Le modèle de migration sans temps d'arrêt dépend d'un espace de stockage suffisant pour créer et exécuter deux environnements complets. Une copie complète des données d'une entreprise est importée dans le nouvel environnement et testée pendant que les employés restent dans l'ancien environnement. Les bogues sont éliminés du nouveau système, garantissant que toutes les applications fonctionnent toujours et que tout est là où il devrait être. Une fois les tests terminés, une nouvelle copie est importée et tous les employés sont transférés dans le nouvel environnement. L'ancien environnement de données est parfois laissé ouvert pendant plusieurs mois afin que les employés puissent obtenir des fichiers de l'ancien système de données, mais pas écrire de nouvelles données sur ces serveurs. Dans toutes les migrations de données, un [audit des données post-migration](/audittrail) est effectué pour vérifier la perte de données.

## Améliorer la migration des données

Une chose qui peut améliorer une migration de données est de nettoyer et de standardiser les pratiques de données avant la migration. L'organisation des données d'une entreprise est souvent le reflet des différentes habitudes de classement de ses employés. Deux personnes ayant le même rôle peuvent très bien utiliser des pratiques complètement différentes. Par exemple, enregistrer les contrats par fournisseur dans un cas et par [exercice](/fiscalyear) et mois dans un autre. L'unification des pratiques en matière de données peut être une tâche beaucoup plus importante que la migration des données, mais des données propres et organisées de manière cohérente, soutenues par des politiques claires, aident à pérenniser les données d'une entreprise pour de nombreuses migrations à venir.

## Points forts

- La migration de données fait référence à la transition d'informations stockées sur un système informatique vers un autre.

- Ce processus peut se produire en raison de mises à niveau technologiques, de déplacements entre des emplacements physiques ou lors d'opérations de maintenance de routine.

- Étant donné que les entreprises dépendent aujourd'hui fortement de leurs données pour mener leurs activités quotidiennes, la migration des données doit être effectuée avec soin et en toute sécurité afin de ne pas provoquer d'interruptions, de perte de données ou de faille de sécurité.
