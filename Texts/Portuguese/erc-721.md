# ERC-721
Os tokens ERC-721 diferem dos tokens ERC-20 no sentido de que os tokens ERC-721 não são fungíveis. Isso significa que cada token é único e, como resultado, não é intercambiável.

## Decompondo o ERC

ERC significa Ethereum Request for Comments. Esta não é uma tecnologia ou plataforma, fornece orientação técnica para desenvolvedores para construção. Em dezembro de 2018, havia nove solicitações finais de comentários do Ethereum. ERC-20, ERC-55, ERC-137, ERC-162, ERC-165, ERC-181, ERC-190, ERC-721 e ERC-1167.

Os desenvolvedores podem criar um ERC enviando uma Proposta de Melhoria Ethereum (EIP).

ERC-721 - O Non-Fungible Token Standard, foi criado por William Entriken, Dieter Shirley, Jacob Evans e Nastassia Sachs devido a incapacidades da interface ERC-20. Através de uma Proposta de Melhoria Ethereum (EIP) criada em 24 de janeiro de 2018.

Um popular aplicativo descentralizado Ethereum, o Cryptokitties, utilizou o ERC-721 para criar colecionáveis digitais exclusivos na forma de gatinhos.

Cada gatinho diferente valia uma quantia diferente no mercado, ditada pelos usuários da plataforma. Cada gatinho diferente comanda um preço diferente e, como resultado, cada token precisa ser único.

A criação de tokens não fungíveis baseados em blockchain permite

- propriedade física (casas, obras de arte e veículos)

- Colecionáveis virtuais (Cryptokitties, pepes raros, cartões colecionáveis)

- Ativos com valor negativo (empréstimos)

Para criar um token ERC-721, o contrato deve estar em conformidade com as interfaces ERC-721 e ERC-165.
