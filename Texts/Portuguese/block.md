# quadra
Em suma, o termo bloco refere-se a arquivos de computador que armazenam dados de transações. Esses blocos são organizados em uma sequência linear que forma uma cadeia interminável de blocos - daí o termo [blockchain](/blockchain).

Assim, todas as informações sobre transações de blockchain são reunidas e registradas dentro desses blocos, e cada bloco recém-gerado é conectado ao anterior através do uso de técnicas [criptográficas .](/cryptography)

A cadeia de blocos vinculados armazena todos os dados de transação gerados desde o lançamento de um determinado blockchain. Assim, os registros vão até o primeiro bloco, que é chamado de bloco zero ou [bloco de gênese](/genesis-block). O número de blocos confirmados desde o bloco de gênese é denotado como a altura do [bloco](/block-height) [k](/block-height).

Tomando o blockchain do Bitcoin como exemplo, os blocos são compostos de vários elementos. Os blocos [Bitcoin](/bitcoin) contêm (entre outras coisas) uma lista de transações recentes, um carimbo de data/hora e uma referência ao bloco que veio antes dele. Essa referência é um hash criptográfico dos dados do bloco anterior.

A geração de um novo bloco sempre inclui o hash do anterior, e é isso que torna os blocos criptograficamente vinculados. Tal estrutura permite a criação de um banco de dados seguro e altamente resistente a adulterações e ataques. O hash do bloco funciona como um identificador. É único para cada bloco e é produzido por meio de um processo chamado [mineração](/mining).

Basicamente falando, o hash do bloco é a solução para um problema matemático complexo, e o minerador que encontra uma solução válida para o próximo bloco recebe o direito de validar esse bloco (e as transações nele). Como a mineração requer uma quantidade significativa de recursos computacionais, os blocos minerados com sucesso produzem novos Bitcoins como forma de recompensar os mineradores por seu trabalho (consulte [recompensa de bloco](/block-reward) ).

Mas, às vezes, dois ou mais mineradores podem encontrar um hash de bloco válido ao mesmo tempo, de modo que dois blocos diferentes sejam transmitidos para a rede. Isso leva à criação de duas cadeias concorrentes. Para resolver esse problema, os participantes da rede ( [nós](/node) ) escolherão a cadeia que eventualmente se tornar a mais longa (com mais “trabalho acumulado”). A outra cadeia é então descartada e seus blocos tornam-se blocos obsoletos (consulte [blocos órfãos](/orphan-block-cryptocurrency) ).

Embora os blocos sejam geralmente discutidos no contexto de transações de criptomoedas, eles também podem estar relacionados a outros tipos de dados digitais armazenados em um sistema blockchain.

##Destaques

- Um bloco refere-se a uma negociação de grande volume que ocorre de uma só vez.

- Às vezes, as negociações em bloco são feitas fora dos mercados abertos para diminuir o impacto no preço do título.

- Para não influenciar os preços de mercado, grandes ordens de bloco podem ser divididas em ordens menores e executadas através de diferentes corretoras para mascarar o tamanho real.

- As bolsas normalmente definem um bloco como mais de 10.000 ações ou uma negociação com valor nocional superior a US$ 200.000.
