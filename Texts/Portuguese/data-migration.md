# Migração de dados
## O que é migração de dados?

A migração de dados é o processo de mover informações digitais armazenadas entre computadores, sistemas ou formatos. A migração de dados ocorre por vários motivos, incluindo substituição ou manutenção de servidor, mudança de [datacenters](/data-warehousing),. projetos de consolidação de dados e atualizações de sistema. Como grande parte do conhecimento corporativo e da inteligência de negócios de uma empresa [está](/business-intelligence-bi) [contida](/business-intelligence-bi) em seus dados, qualquer projeto de migração de dados deve ser feito com cuidado para minimizar os riscos.

## Entendendo a migração de dados

A migração de dados representa um risco significativo para a [continuidade dos negócios](/business-continuity-planning) se não for feita corretamente. A perda de dados é, obviamente, o pior cenário possível, mas as empresas também devem lidar com o tempo de inatividade, problemas de compatibilidade e problemas gerais de desempenho do sistema. A migração de dados é dificultada por grandes quantidades de dados, diversos formatos de dados e hábitos de dados divergentes dentro de uma corporação.

Para minimizar os riscos da migração de dados, as empresas criam políticas detalhadas de migração de dados que priorizam [backups](/back-up),. ordem de movimentação e ambientes de dados simultâneos quando possível. Se não for possível para uma empresa executar um ambiente de pré-migração enquanto o novo ambiente estiver preparado, haverá um tempo de inatividade significativo, pois as operações de negócios nos aplicativos atuais serão suspensas para permitir a migração de dados. Esse tipo de migração de dados de parada, transferência e início pode ser necessário ao migrar para novas plataformas ou quando há limites rígidos no armazenamento físico e trocas ou correções são necessárias na tecnologia de armazenamento existente.

## Migração de dados com tempo de inatividade zero

O modelo de migração sem tempo de inatividade depende de ter armazenamento suficiente para criar e executar dois ambientes completos. Uma cópia completa dos dados de uma empresa é levada para o novo ambiente e testada enquanto os funcionários permanecem no ambiente antigo. Os bugs são resolvidos do novo sistema, garantindo que todos os aplicativos ainda funcionem e tudo esteja onde deveria estar. Após a conclusão do teste, uma nova cópia é trazida e todos os funcionários são transferidos para o novo ambiente. O ambiente de dados antigo às vezes é deixado aberto por um período de meses para que os funcionários possam obter arquivos do sistema de dados antigo, mas não gravar novos dados nesses servidores. Em todas as migrações de dados, é feita uma [auditoria de dados](/audittrail) pós-migração para verificar a perda de dados.

## Como melhorar a migração de dados

Uma coisa que pode melhorar uma migração de dados é limpar e padronizar as práticas de dados antes da migração. A organização dos dados de uma empresa geralmente é um reflexo dos diferentes hábitos de arquivamento de seu pessoal. Duas pessoas com o mesmo papel podem muito bem usar práticas completamente diferentes. Por exemplo, salvar contratos por fornecedor em um caso e por [ano fiscal](/fiscalyear) e mês em outro. Unificar as práticas de dados pode ser uma tarefa muito maior do que realmente migrar os dados, mas dados limpos e organizados de forma coerente, apoiados por políticas claras, ajudam a proteger os dados de uma empresa no futuro para muitas migrações ainda por vir.

## Destaques

- A migração de dados refere-se à transição de informações armazenadas em um sistema de computador para outro.

- Esse processo pode ocorrer devido a atualizações de tecnologia, movimentação entre locais físicos ou durante operações de manutenção de rotina.

- Como as empresas hoje dependem muito de seus dados para realizar negócios diários, a migração de dados deve ser realizada com cuidado e segurança para não causar interrupções, perda de dados ou violação de segurança.
