# Process Value Analysis (PVA)
## Vad är processvärdeanalys?

Process Value Analysis (PVA) är undersökningen av en intern process som företag genomför för att avgöra om den kan effektiviseras. PVA tittar på vad [kunden](/customer) vill ha och frågar sedan om ett steg i en process är nödvändigt för att uppnå det resultatet. Målet med PVA är att eliminera onödiga steg och utgifter som uppstår i [värdekedjan som](/valuechain) krävs för att skapa en vara eller tjänst utan att offra kundnöjdheten. Resultatet är att en vara eller tjänst kan levereras till kunden snabbare och till en lägre kostnad.

## Förstå Process Value Analysis (PVA)

Vid genomförandet av PVA kommer chefer att överväga om någon ny teknik skulle kunna implementeras lönsamt, om det görs fel som kan undvikas, om det finns extra steg i processen som är onödiga, och så vidare. Eventuella steg i processen som identifieras som inte tillför [ekonomiskt värde](/economic-value) kan ändras eller kastas bort. En process kan upprepade gånger granskas när ny teknik dyker upp som kan göra processen mer effektiv.

> En PVA kräver att chefer objektivt analyserar alla områden av sin verksamhet och pekar ut de aktiviteter som inte tillför värde och inte är kostnadseffektiva.

>

Ledningen kan genomföra PVA:er för att granska och utvärdera processer inom en hel rad affärsområden i hela företaget. Till exempel kan ett företag bedöma processerna och funktionen för inkommande [logistik](/logistics),. drift, utgående logistik, marknadsföring, försäljning och kundservice.

## Kritik av processvärdeanalys (PVA)

En risk med PVA är att vissa kritiska steg i en process kan elimineras. Processer inkluderar ibland kontrollpunkter för att säkerställa att reglerna följs. Dessa regler kan utformas för att upprätta [kostnadskontroller](/cost-control),. säkerställa korrekta redovisningsförfaranden och andra interna kontroller. Att eliminera en nödvändig kontrollpunkt kan resultera i oavsiktliga konsekvenser för företaget.

Till exempel, om en PVA fokuserar för mycket på att minska kostnaderna, löper företaget risken att ta bort eller ändra en process som håller verksamheten igång smidigt. En illustration av detta skulle vara ett företag som bestämmer sig för att [lägga ut](/outsourcing) sin kundtjänstavdelning till en tredjepartsleverantör bara för att upptäcka att säljaren inte har arbetsstyrkan eller expertis på plats för att ge bra kundservice. För att förhindra att situationer som denna uppstår kan ett företag anlita en kontrollanalytiker för att hjälpa till att övervaka PVA och för att rådgöra med den interna redovisningspersonalen och ledningen.

## Exempel på processvärdeanalys (PVA)

Vissa företag har åtagit sig PVA för att effektivisera sina inköpsprocesser. För små inköp har de valt att ge [chefsupphandlingskort](/procurement) från stora kreditkortsföretag. Detta har visat sig vara billigare än att kräva att små inköp går igenom den flerstegsprocess som normalt krävs för stora inköp.

Företag kommer ibland att genomföra PVAs när de har gjort ett [förvärv](/acquisition). En PVA kan avslöja om det förvärvade företaget har processer som är mindre effektiva än det förvärvande företagets eller vice versa. En PVA kan också hjälpa ledningen att maximera de [synergier](/synergy) eller potentiella ekonomiska fördelar som den hoppas uppnå genom att slå samman företag.

##Höjdpunkter

- En nackdel med att genomföra en PVA är potentialen som företaget eliminerar en process eller ändrar ett steg som sedan resulterar i oavsiktliga konsekvenser, som att försena leveransen av varor eller försämra relationen med sina kunder.

– Företag som genomför en PVA vill tillhandahålla varor eller tjänster till kunder till en lägre kostnad och snabbare.

- En Process Value Analysis (PVA) undersöker varje steg i en specifik affärsprocess för att avgöra om den kan förbättras eller effektiviseras samtidigt som kundnöjdheten bibehålls.
