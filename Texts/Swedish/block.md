# blockera
Kortfattat hänvisar termen block till datorfiler som lagrar transaktionsdata. Dessa block är ordnade i en linjär sekvens som bildar en ändlös kedja av block - därav termen [blockchain](/blockchain).

Så all information om blockchain-transaktioner samlas in och registreras i dessa block, och varje nygenererat block är kopplat till det föregående genom användning av [kryptografiska](/cryptography) tekniker.

Kedjan av länkade block lagrar all transaktionsdata som genererats sedan lanseringen av en viss blockkedja. Så posterna går hela vägen tillbaka till det första blocket, som kallas block noll eller [genesis block](/genesis-block). Antalet bekräftade block sedan genesisblocket betecknas som [block](/block-height) [k-höjden](/block-height).

Om man tar Bitcoin-blockkedjan som exempel, består block av flera element. [Bitcoin-](/bitcoin) block innehåller (bland annat) en lista över senaste transaktioner, en tidsstämpel och en referens till blocket som kom precis innan det. Denna referens är en kryptografisk hash av föregående blocks data.

Genereringen av ett nytt block inkluderar alltid hashen från det föregående, och det är detta som gör blocken kryptografiskt länkade. En sådan struktur möjliggör skapandet av en säker databas som är mycket motståndskraftig mot manipulering och attacker. Blockhashen fungerar som en identifierare. Det är unikt för varje block och produceras genom en process som kallas [gruvdrift](/mining).

I grund och botten är blockhash lösningen på ett komplext matematiskt problem, och gruvarbetaren som hittar en giltig lösning för nästa block får rätten att validera det blocket (och transaktionerna däri). Eftersom gruvdrift kräver en betydande mängd beräkningsresurser, producerar framgångsrikt minerade block nya Bitcoins som ett sätt att belöna gruvarbetare för deras arbete (se [blockbelöning](/block-reward) ).

Men ibland kan två eller flera gruvarbetare hitta en giltig blockhash samtidigt, så att två olika block sänds till nätverket. Detta leder till skapandet av två konkurrerande kedjor. För att lösa detta problem kommer deltagarna i nätverket ( [noder](/node) ) att välja den kedja som så småningom blir längst (med mer "ackumulerat arbete"). Den andra kedjan kasseras sedan, och dess block blir inaktuella block (se [föräldralösa block](/orphan-block-cryptocurrency) ).

Även om block vanligtvis diskuteras i samband med kryptovalutatransaktioner, kan de också vara relaterade till andra typer av digital data som lagras i ett blockkedjesystem.

##Höjdpunkter

– Ett block avser en stor volymhandel som sker på en gång.

- Blockaffärer görs ibland utanför de öppna marknaderna för att minska påverkan på värdepapprets pris.

– För att inte påverka marknadspriserna kan stora blockorder delas upp i mindre order och utföras via olika mäklare för att maskera den verkliga storleken.

- Börser definierar vanligtvis ett block som mer än 10 000 aktier eller en handel som har ett nominellt värde på över 200 000 USD.
