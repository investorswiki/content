# HyperText Markup Language – HTML
## Vad är HyperText Markup Language (HTML)?

HyperText Markup Language (HTML) är uppsättningen av uppmärkningssymboler eller koder som infogas i en fil som är avsedd att visas på Internet. Markeringen talar om för webbläsare hur de ska visa en webbsidas ord och bilder.

Varje enskild uppmärkningskod (som skulle falla mellan "<" och ">"-tecken) kallas ett element, även om många människor också hänvisar till det som en tagg. Vissa element kommer i par som indikerar när någon visningseffekt ska börja och när den ska sluta.

## HTML förklaras

HyperText Markup Language är datorspråket som underlättar skapandet av webbplatser. Språket, som har kodord och syntax precis som vilket annat språk som helst, är relativt lätt att förstå och, allt eftersom tiden går, allt kraftfullare i vad det tillåter någon att skapa. HTML fortsätter att utvecklas för att möta kraven och kraven på Internet under sken av World Wide Web Consortium, organisationen som designar och underhåller språket; till exempel med övergången till [Web 2.0](/web-20).

HyperText är metoden med vilken Internetanvändare navigerar på webben. Genom att klicka på specialtext som kallas hyperlänkar förs användare till nya sidor. Användningen av hyper betyder att den inte är linjär, så användare kan gå var som helst på Internet genom att helt enkelt klicka på de tillgängliga länkarna. Markup är vad HTML-taggar gör med texten inuti dem; de markerar det som en specifik typ av text. Till exempel kan uppmärkningstext komma i form av fetstil eller kursiv stil för att dra särskild uppmärksamhet till ett ord eller en fras.

## HyperText Markup Language Basics

I grunden är HTML en serie korta koder som skrivs in i en textfil. Det här är taggarna som driver HTMLs möjligheter. Texten sparas som en HTML-fil och visas via en webbläsare. Webbläsaren läser filen och översätter texten till en synlig form, enligt anvisningarna från de koder som författaren använde för att skriva vad som blir den synliga återgivningen. Att skriva HTML kräver att taggar används korrekt för att skapa författarens vision.

Taggarna är det som skiljer normal text från HTML-kod. Taggar är orden mellan vad som kallas vinkelparenteser, som tillåter grafik, bilder och tabeller att visas på webbsidan. Olika taggar har olika funktioner. De mest grundläggande taggarna tillämpar formatering på text. Eftersom webbgränssnitten behöver bli mer dynamiska kan Cascading Style Sheets (CSS) och JavaScript-applikationer användas. CSS gör webbsidor mer tillgängliga och JavaScript ger kraft till grundläggande HTML.

## HTML vs. XML

Till skillnad från HTML, tillåter Extensible Markup Language eller XML, användare att definiera sin egen uppmärkning. Med till exempel XML kan en användare välja att beteckna en fotnot med taggen <footnote>, medan en annan användare kan välja <fn>.

Med HTML kan endast en förutbestämd tagg användas för att beteckna en specifik typ av information. XML-dokument är tänkta att vara lätta att läsa eftersom de innehåller användardefinierade taggar och eftersom dokumenten endast består av uppmärkning och innehåll.

## Höjdpunkter

- HyperText Markup Language (HTML) är det grundläggande skriptspråket som används av webbläsare för att rendera sidor på world wide web.

- Tidiga versioner av HTML var statiska (Web 1.0), medan nyare iterationer har en hel del dynamisk flexibilitet (Web 2.0, 3.0).

- Markup är texten som visas mellan två spetsiga parenteser (t.ex. <fotnot>), och innehåll är allt annat.

- HyperText låter en användare klicka på en länk och omdirigeras till en ny sida som den länken refererar till.
