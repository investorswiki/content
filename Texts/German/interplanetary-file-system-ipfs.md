# Interplanetares Dateisystem (IPFS)
InterPlanetary File System (IPFS) ist ein Open-Source-Projekt, das ein Peer-to-Peer-Dateisystem aufbaut. Das IPFS-Projekt ist ein ehrgeiziges Projekt, das die Art und Weise, wie wir das Internet nutzen, möglicherweise verändern kann. Sein endgültiges Ziel ist es, alle Geräte zu verbinden, die dasselbe Dateisystem haben, aber nicht auf die gleiche Weise, wie es das Web heute tut. Um die Grundlagen von IPFS zu verstehen, ist es hilfreich, es mit HTTP zu vergleichen.

Das aktuelle World Wide Web basiert im Wesentlichen auf den Protokollen HTTP und HTTPS. Kurz gesagt handelt es sich um Anwendungsprotokolle, die eine weltweite Datenkommunikation und -zugänglichkeit ermöglichen.

HTTP funktioniert als Request-Response-Protokoll, das Benutzer (Clients) basierend auf ihrem Standort mit Servern verbindet. Wenn Alice beispielsweise auf eine Website geht, fordert ihr Webbrowser den Inhalt beim Hosting-Server (z. B. Amazon Web Services) an. Wenn alles gut geht, sendet der AWS-Server die Webseiten an sie zurück.

Der Inhalt ist jedoch nicht dauerhaft, da er von einem zentralen Server verwaltet wird. Und da die AWS eine große Anzahl von Websites hostet, fällt ein großer Prozentsatz des Internets mit aus, wenn ihre Server aus irgendeinem Grund ausfallen.

Im Gegensatz dazu ermöglicht IPFS die Schaffung eines permanenten und verteilten Webs, in dem alle Arten von digitalen Daten gespeichert und geteilt werden können. Während die HTTP-Kommunikation auf einen Server (mit einem bestimmten Standort) angewiesen ist, um den Inhalt bereitzustellen, konzentriert sich die IPFS-Kommunikation auf den Inhalt selbst.

Mit IPFS würde Alice den Inhalt nicht mehr direkt beim AWS-Server abfragen. Stattdessen würde sie das verteilte Netzwerk fragen: „Wer kann mir diesen Inhalt bereitstellen?“ und die engsten Kollegen würden schnell antworten.

Abhängig von der Implementierung kann IPFS im Vergleich zu HTTP viele Vorteile bieten, z. B. Zensurresistenz, Datenintegrität, niedrigere Betriebskosten, bessere Leistung und Sicherheit.

Einige der Einschränkungen hängen mit den geringen Anreizen zur Teilnahme am Netzwerk zusammen, was zu einer kleinen Population von Peers führt. Die eingeschränkte Akzeptanz erschwert die dauerhafte Verfügbarkeit von Dateien. Wenn bestimmte Daten nur von einer Handvoll Knoten gehostet werden und alle offline gehen, werden sie unzugänglich.
