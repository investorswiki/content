# Bibliothek
In der Computerprogrammierung bezieht sich der Begriff Bibliothek auf eine gut dokumentierte Sammlung stabiler Ressourcen, die ausführbare Dateien, Dokumentation, Nachrichtenvorlagen und geschriebenen Code enthalten können.

Der Begriff kann sich auch auf eine Sammlung vorgefertigter Module beziehen, die bei Verwendung eine bestimmte Aktion ausführen oder eine bestimmte Ausgabe erzeugen. Die Module in der Bibliothek können die Form von Code, Klassendefinitionen, Prozeduren, Skripten und Konfigurationsdaten annehmen. Das bedeutet, dass Entwickler die Module in der Bibliothek aufrufen können, ohne den gesamten Code für jedes Modul von Grund auf neu schreiben zu müssen. Mit anderen Worten, sie können das von anderen Bibliotheken bereitgestellte Verhalten wiederverwenden, anstatt selbst neue Module zu erstellen.

Bibliotheken können für den nicht spezialisierten Gebrauch verallgemeinert werden (z. B. Zeitangaben, grundlegende mathematische Operationen) oder sehr spezifisch für einen bestimmten Anwendungsfall (z. B. Dekodierung der Komprimierung von Tondateien).

Daher bringt die Verwendung von Codebibliotheken viele Vorteile für Entwickler und Informatiker. Einige von ihnen beinhalten die effizientere Nutzung von Zeit und Ressourcen, was es Entwicklern erleichtert, Software zu entwickeln und auszuführen.

Ein Nachteil der Verwendung von Code-Bibliotheken ist der sogenannte Black-Box-Effekt, bei dem der Entwickler nur die Ein- und Ausgabe seiner Verwendung sehen kann, was bedeutet, dass er oft wenig oder gar nicht versteht, was dazwischen passiert.
