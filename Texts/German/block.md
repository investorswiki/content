# Block
Kurz gesagt bezieht sich der Begriff Block auf Computerdateien, die Transaktionsdaten speichern. Diese Blöcke sind in einer linearen Folge angeordnet, die eine endlose Kette von Blöcken bildet – daher der Begriff [Blockchain](/blockchain).

Daher werden alle Informationen über Blockchain-Transaktionen in diesen Blöcken gesammelt und aufgezeichnet, und jeder neu generierte Block wird durch den Einsatz [kryptografischer](/cryptography) Techniken mit dem vorherigen verbunden.

Die Kette verknüpfter Blöcke speichert alle Transaktionsdaten, die seit dem Start einer bestimmten Blockchain generiert wurden. Die Aufzeichnungen gehen also bis zum ersten Block zurück, der als Block Null oder [Genesis-Block bezeichnet wird](/genesis-block). Die Anzahl der bestätigten [Blöcke seit dem Entstehungsblock wird als](/block-height) [Blockhöhe](/block-height) bezeichnet .

Am Beispiel der Bitcoin-Blockchain bestehen Blöcke aus mehreren Elementen. [Bitcoin-](/bitcoin) Blöcke enthalten (unter anderem) eine Liste der letzten Transaktionen, einen Zeitstempel und einen Verweis auf den Block, der direkt davor kam. Diese Referenz ist ein kryptografischer Hash der Daten des vorherigen Blocks.

Die Generierung eines neuen Blocks beinhaltet immer den Hash des vorherigen, und dadurch werden die Blöcke kryptografisch verknüpft. Eine solche Struktur ermöglicht die Erstellung einer sicheren Datenbank, die sehr widerstandsfähig gegen Manipulationen und Angriffe ist. Der Block-Hash wirkt wie ein Identifikator. Es ist für jeden Block einzigartig und wird durch einen Prozess namens [Mining hergestellt](/mining).

Grundsätzlich ist der Block-Hash die Lösung für ein komplexes mathematisches Problem, und der Miner, der eine gültige Lösung für den nächsten Block findet, erhält das Recht, diesen Block (und die darin enthaltenen Transaktionen) zu validieren. Da das Mining eine erhebliche Menge an Rechenressourcen erfordert, produzieren erfolgreich geminte Blöcke neue Bitcoins, um Miner für ihre Arbeit zu belohnen (siehe [Blockbelohnung](/block-reward) ).

Aber manchmal finden zwei oder mehr Miner gleichzeitig einen gültigen Block-Hash, sodass zwei verschiedene Blöcke an das Netzwerk gesendet werden. Dies führt zur Bildung von zwei konkurrierenden Ketten. Um dieses Problem anzugehen, wählen die Teilnehmer des Netzwerks ( [Knoten ) die Kette, die schließlich die längste wird (mit mehr „kumulierter Arbeit“).](/node) Die andere Kette wird dann verworfen und ihre Blöcke werden zu veralteten Blöcken (siehe [verwaiste Blöcke](/orphan-block-cryptocurrency) ).

Obwohl Blöcke normalerweise im Zusammenhang mit Kryptowährungstransaktionen diskutiert werden, können sie sich auch auf andere Arten von digitalen Daten beziehen, die auf einem Blockchain-System gespeichert sind.

## Höhepunkte

- Ein Block bezieht sich auf einen großvolumigen Handel, der gleichzeitig stattfindet.

- Block Trades werden manchmal außerhalb der offenen Märkte durchgeführt, um die Auswirkungen auf den Kurs des Wertpapiers zu verringern.

- Um die Marktpreise nicht zu beeinflussen, können große Blockorders in kleinere Orders aufgeteilt und über verschiedene Broker ausgeführt werden, um die wahre Größe zu verschleiern.

- Börsen definieren einen Block in der Regel als mehr als 10.000 Aktien oder einen Handel mit einem Nennwert von über 200.000 USD.
