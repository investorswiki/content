# Füllen oder töten - FOK
Einige Börsen und Handelsplattformen bieten eine Orderart an, die als „Fill or Kill Order“ (FOK) bekannt ist. Der Begriff bezieht sich auf die Idee, dass eine Order sofort vollständig ausgeführt oder gar nicht ausgeführt werden muss. Er ist eng verwandt mit dem Auftragstyp [„Alles oder Nichts“ (A](/aon) [ON)](/aon),. der sich auf einen Auftrag bezieht, der vollständig oder gar nicht ausgeführt werden muss. Im Gegensatz zu FOK-Aufträgen haben AON-Aufträge jedoch keinen konkreten Fokus auf den unmittelbaren Zeitpunkt.

Fill-or-Kill-Orders werden häufig verwendet, wenn ein Händler keine Teillieferung von Vermögenswerten akzeptieren möchte. Zum Beispiel, wenn sie eine zeitbasierte Nachfrage haben, um ihre Aufträge auf unterschiedlichen und nicht verbundenen Märkten oder Börsen auszuführen. Eine FOK-Order würde es ihnen also ermöglichen, mehrere Orders zu erstellen und darauf zu warten, dass eine vollständig ausgeführt wird, ohne das Risiko einzugehen, Teilausführungen zu erhalten. Nachdem eine der Orders vollständig ausgeführt wurde, kann der Händler die verbleibenden Orders stornieren.

Betrachten Sie als Beispiel für einen Fill-or-Kill-Anwendungsfall das folgende Szenario:

Alice möchte einen [Altcoin einrichten](/altcoin) [Masternode](/masternode) sofort, aber eine der Voraussetzungen für den Betrieb einer Masternode ist, dass sie 1000 Einheiten dieser bestimmten Kryptowährung halten muss. Wenn die Zeit kein limitierender Faktor wäre, könnte Alice zahlreiche Kaufaufträge erteilen, bis schließlich die Schwelle von 1000 erreicht wird. Da sie jedoch möchte, dass der Masternode ohne allzu große Verzögerung einsatzbereit ist, kann sie mehrere Fill-or-Kill-Kaufaufträge für 1000 Einheiten der Altcoin (an verschiedenen Börsen) erteilen. Auf diese Weise zahlt Alice nur dann für die Altcoins, wenn sie die gewünschten 1.000 Einheiten erhält, und dies ermöglicht ihr, jede nicht vollständig ausgeführte Bestellung zu stornieren.

## Höhepunkte

- Eine Fill-or-Kill-Order (FOK) ist eine Order, die sofort zum Markt oder zu einem bestimmten Preis ausgeführt oder storniert wird, wenn sie nicht ausgeführt wird.

- Typische FOK-Orders dauern ein paar Sekunden, um Störungen des Aktienkurses zu minimieren, und Teilfüllungen sind nicht zulässig.

- Eine FOK-Order kombiniert eine All-or-None (AON)-Spezifikation, die angibt, dass sie vollständig mit einem Immediate-or-Cancel (IOC)-Zeitrahmen ausgeführt werden muss.
