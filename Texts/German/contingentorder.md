# Bedingte Bestellung
## Was ist eine bedingte Bestellung?

Ein bedingter Auftrag ist ein Auftrag, der mit der Ausführung eines anderen Ereignisses verbunden ist und dessen Ausführung erfordert. Der bedingte Auftrag wird aktiv oder ausgeführt, wenn das Ereignis eintritt. Ein Beispiel ist eine [Stop-Loss-Order](/stop-lossorder). Der Stop-Loss-Verkauf hängt davon ab, ob ein Wertpapier zuerst gekauft wird.

Eine bedingte Bestellung ist eine Art [bedingte Bestellung](/conditionalorder).

## Den bedingten Auftrag verstehen

Kontinentale Bestellungen sind davon abhängig, dass etwas anderes passiert, bevor die bedingte Bestellung bearbeitet wird.

Eine bedingte Bestellung kann sein:

1. Ein Auftrag, der die gleichzeitige Ausführung von zwei oder mehr Transaktionen beinhaltet.

1. Eine Order, deren Ausführung von der Ausführung einer anderen Order abhängt.

1. Eine Order, bei der die Ausführung von bestimmten Kriterien abhängt, wie z. B. Preis, Volumen, Zeit oder andere Faktoren, die erfüllt werden.

Bei einer gleichzeitigen Transaktion sind die Aufträge voneinander abhängig, da alle Aufträge gleichzeitig bearbeitet werden müssen. Wenn sie nicht gleichzeitig bearbeitet werden können, bleiben die Aufträge offen, bis sie gleichzeitig ausgeführt werden können.

Eine bedingte Bestellung könnte auch auf einer anderen Bestellung oder einem anderen Ereignis beruhen. Beispielsweise kann ein Händler einen Kaufauftrag für [Optionen](/option) davon abhängig machen, dass er im Rahmen eines Aktienkaufauftrags ausgeführt wird. Erst wenn sie die Aktie besitzen, sollte die Optionsorder ausgeführt werden. Der Händler könnte auch verlangen, dass die Aufträge gleichzeitig ausgeführt werden.

Kriterien, von denen eine Order abhängig ist, können auch [Volumen](/volume),. Preis, Zeit oder eine Vielzahl anderer grundlegender oder [technischer](/technicalindicator) Tools umfassen. Beispielsweise kann eine Order davon abhängen, dass ein Wertpapier einen bestimmten Preis erreicht, ein bestimmtes Volumen aufweist und beides innerhalb bestimmter Stunden des Tages erreicht.

## Eventuelle Auftragsverwendungen

Bedingte Orders sind nützlich, da sie es einem Trader ermöglichen, eine Strategie oder mehrere Positionen zu implementieren, sobald das ursprüngliche Ereignis eintritt. Wenn ein Händler jede Transaktion in sequentieller Reihenfolge buchen müsste, könnte er anfällig für Verluste oder Preisänderungen sein. Angenommen, ein Händler möchte eine Aktie zu 50 $ kaufen, aber er möchte auch einen Stop-Loss bei 49,85 $ und eine Verkaufsorder ( [Ziel](/pricetarget) ) bei 50,30 $ platzieren, sobald er die Aktie besitzt. Dies wird als [Klammerreihenfolge bezeichnet](/bracketedbuyorder).

Sie könnten dies manuell tun, aber wenn sich der Aktienkurs sehr schnell bewegt, erhalten sie ihre Aufträge möglicherweise nicht rechtzeitig. Wenn der Stop-Loss und das Ziel abhängig von der Ausführung der Kauforder gesendet werden, weiß der Händler, dass er Orders hat, um Gewinne zu erzielen und das Risiko zu kontrollieren, unabhängig davon, ob sich die Aktie nach oben oder unten bewegt.

Außerdem gibt es ein weiteres Problem bei der manuellen Ausführung. Wenn Sie eine Aktie kaufen und versuchen, eine Verkaufsorder über dem Preis und unter dem Preis (Ziel und Stop-Loss) zu platzieren, lässt die Software dies wahrscheinlich nicht zu. Dies liegt daran, dass die Handelsplattform denkt, dass Sie versuchen, zweimal zu verkaufen. Wenn der Stop-Loss und das Ziel jedoch von der Ausgangsposition abhängen, weiß es, dass bei Erreichen des Stop-Loss oder des Ziels diese Order die Position [schließt](/closeposition) und die andere Order storniert werden kann, da die Ausgangsposition nicht mehr existiert. Die Software weiß, dass Sie nicht versuchen, zweimal zu verkaufen, und lässt Sie daher beide Bestellungen gleichzeitig aufgeben. Eine der bedingten Bestellungen wird in diesem Fall storniert, wenn die andere bedingte Bestellung ausgeführt wird.

Die meisten Broker bieten Contingent Order-Funktionalität an. Dies kann durch verschiedene Auftragsarten geschehen, wie z. B. [Korbaufträge](/baskettrade),. Multi [-Leg-](/leg) Optionsaufträge oder Klammeraufträge. Der Händler gibt zuerst ein, was geschehen soll, und legt dann die Parameter für die bedingte(n) Order(s) fest. Bei einer Klammerbestellung (die bedingte Bestellungen erstellt) würde dies bedeuten, die Erstbestellung aufzugeben. Dann werden der Stop-Loss und das Ziel festgelegt. Die Stop-Loss- und Target-Orders werden nur eingesetzt, wenn die ursprüngliche Order ausgeführt wird.

## Kontingent-Order-Beispiel im Optionsmarkt

Bedingte Aufträge werden häufig auf dem Optionsmarkt verwendet, da einige Optionsgeschäfte mehrere Beine haben.

Beispielsweise beinhaltet eine [Buy-Write-](/buy-write) Strategie den gleichzeitigen Kauf einer Long-Aktienposition und das Schreiben einer [Call-Option](/calloption) gegen diese Position. Ein Trader kann eine Kauforder für die Long-Aktienposition platzieren, die davon abhängig ist, dass die Call-Option [zu](/writing-an-option) einem bestimmten Preis verkauft wird, oder umgekehrt.

Ohne einen bedingten Auftrag müssten Händler zwei separate [Transaktionen ausführen](/transaction). Wenn sich der Preis zwischen den beiden Aufträgen ändert, könnte der Händler einen Moment erhöhten Risikos oder möglicherweise einen schlechteren (oder besseren) Preis als erwartet bei einer der Transaktionen erleben. Die bedingte Order stellt sicher, dass die Position genau so eröffnet wird, wie der Händler es erwartet.

## Höhepunkte

- Orders können von einer anderen Order oder einem anderen Ereignis abhängen, z. B. wenn ein Stop-Loss automatisch gesendet wird, sobald ein Trade eingegeben wurde.

- Ein bedingter Auftrag ist ein Auftrag, der auf das Eintreten eines bestimmten Ereignisses angewiesen ist, bevor er aktiviert werden kann.

- Aufträge können voneinander abhängig sein, z. B. wenn zwei oder mehr Aufträge gleichzeitig ausgeführt werden müssen.
