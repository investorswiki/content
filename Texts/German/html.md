# HyperText-Auszeichnungssprache – HTML
## Was ist HyperText Markup Language (HTML)?

HyperText Markup Language (HTML) ist ein Satz von Markup-Symbolen oder -Codes, die in eine Datei eingefügt werden, die zur Anzeige im Internet bestimmt ist. Das Markup teilt Webbrowsern mit, wie die Wörter und Bilder einer Webseite angezeigt werden sollen.

Jeder einzelne Markup-Code (der zwischen den Zeichen „<“ und „>“ liegen würde) wird als Element bezeichnet, obwohl viele ihn auch als Tag bezeichnen. Einige Elemente kommen paarweise vor, die angeben, wann ein Anzeigeeffekt beginnen und wann er enden soll.

## HTML erklärt

HyperText Markup Language ist die Computersprache, die die Erstellung von Websites erleichtert. Die Sprache, die wie jede andere Sprache über Codewörter und eine Syntax verfügt, ist relativ einfach zu verstehen und wird im Laufe der Zeit immer leistungsfähiger in dem, was sie jemandem erlaubt, etwas zu erschaffen. HTML entwickelt sich weiter, um die Anforderungen und Anforderungen des Internets unter dem Deckmantel des World Wide Web Consortium zu erfüllen, der Organisation, die die Sprache entwirft und pflegt; zum Beispiel beim Übergang zu [Web 2.0](/web-20).

HyperText ist die Methode, mit der Internetnutzer im Web navigieren. Durch Klicken auf einen speziellen Text namens Hyperlinks werden Benutzer auf neue Seiten gebracht. Die Verwendung von Hyper bedeutet, dass es nicht linear ist, sodass Benutzer überall im Internet hingehen können, indem sie einfach auf die verfügbaren Links klicken. Markup ist das, was HTML-Tags mit dem darin enthaltenen Text machen; sie kennzeichnen es als eine bestimmte Art von Text. Markup-Text könnte beispielsweise fett oder kursiv gedruckt werden, um besondere Aufmerksamkeit auf ein Wort oder einen Satz zu lenken.

## Grundlagen der HyperText Markup Language

Im Kern ist HTML eine Reihe von kurzen Codes, die in eine Textdatei eingegeben werden. Dies sind die Tags, die die Fähigkeiten von HTML antreiben. Der Text wird als HTML-Datei gespeichert und über einen Webbrowser angezeigt. Der Browser liest die Datei und übersetzt den Text in eine sichtbare Form, wie von den Codes vorgegeben, die der Autor verwendet hat, um zu schreiben, was das sichtbare Rendering wird. Beim Schreiben von HTML müssen Tags korrekt verwendet werden, um die Vision des Autors zu erstellen.

Die Tags trennen normalen Text vom HTML-Code. Tags sind Wörter zwischen sog. spitzen Klammern, die das Erscheinen von Grafiken, Bildern und Tabellen auf der Webseite ermöglichen. Unterschiedliche Tags führen unterschiedliche Funktionen aus. Die einfachsten Tags wenden eine Formatierung auf Text an. Da Webschnittstellen dynamischer werden müssen, können Cascading Style Sheets (CSS) und JavaScript-Anwendungen verwendet werden. CSS macht Webseiten zugänglicher und JavaScript verleiht grundlegendem HTML mehr Leistung.

## HTML vs. XML

Im Gegensatz zu HTML ermöglicht Extensible Markup Language oder XML Benutzern, ihr eigenes Markup zu definieren. Unter Verwendung von XML könnte beispielsweise ein Benutzer eine Fußnote mit dem Tag <footnote> kennzeichnen, während ein anderer Benutzer sich für <fn> entscheiden könnte.

Bei Verwendung von HTML kann nur ein vordefiniertes Tag verwendet werden, um einen bestimmten Informationstyp zu bezeichnen. XML-Dokumente sollen leicht lesbar sein, da sie benutzerdefinierte Tags enthalten und die Dokumente nur aus Markup und Inhalt bestehen.

## Höhepunkte

- HyperText Markup Language (HTML) ist die grundlegende Skriptsprache, die von Webbrowsern verwendet wird, um Seiten im World Wide Web darzustellen.

- Frühere Versionen von HTML waren statisch (Web 1.0), während neuere Iterationen ein hohes Maß an dynamischer Flexibilität aufweisen (Web 2.0, 3.0).

- Markup ist der Text, der zwischen zwei spitzen Klammern steht (z. B. <Fußnote>), und Inhalt ist alles andere.

- HyperText ermöglicht es einem Benutzer, auf einen Link zu klicken und zu einer neuen Seite umgeleitet zu werden, auf die von diesem Link verwiesen wird.
