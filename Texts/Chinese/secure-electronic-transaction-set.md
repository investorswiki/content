# 安全电子交易 (SET)
## 什么是安全电子交易 (SET)？

安全电子交易 (SET) 是电子商务网站用于保护电子借记卡和信用卡支付的早期通信协议。安全电子交易被用于通过互联网上的电子门户促进消费者卡信息的安全传输。安全的电子交易协议负责屏蔽卡信息的个人详细信息，从而防止商家、黑客和电子窃贼访问消费者信息。

## 了解安全电子交易 (SET)

大多数主要的电子交易提供商都支持安全电子交易协议，例如 Visa 和 MasterCard。这些协议允许商家在没有实际看到的情况下验证其客户的卡信息，从而保护客户。卡上的信息直接转移到信用卡公司进行验证。

安全电子交易过程使用数字证书，这些证书被分配以提供对资金的电子访问，无论是信用额度还是银行账户。每次以电子方式进行购买时，都会为交易参与者（客户、商家和金融机构）生成一个加密的数字证书，以及匹配的数字密钥，使他们能够确认对方的证书并验证交易。使用的算法将确保只有拥有相应数字密钥的一方才能确认交易。因此，消费者的信用卡或银行账户信息可用于完成交易，而无需透露他们的任何个人详细信息，例如他们的帐号。安全的电子交易旨在成为防止账户盗窃、黑客攻击和其他犯罪行为的一种安全形式。

## 安全电子交易的历史

安全电子交易协议的发展是对电子商务交易的出现和增长的回应，特别是消费者驱动的互联网购买。在线开展业务是 1990 年代中期的一种新现象。同样，可用于保护这些交易的安全措施仍在发展中，并且在不同程度上有效。安全电子交易标准定义的协议允许零售商和金融机构使用在线支付系统，因为它们拥有适当的软件来正确解密和处理数字交易。 1996 年，由 VISA 和 Mastercard 与 GTE、IBM、Microsoft、Netscape、SAIC、Terisa Systems、RSA 和 VeriSign 合作组成的 SET 联盟确立了结合不兼容的安全协议（Visa 和 Microsoft 的 STT）的目标。 ; 来自 Mastercard 和 IBM 的 SEPP）成为一个单一的标准。

引入安全电子交易定义的协议后，出现了其他用于在线借记卡和信用卡交易的数字安全标准。 Visa 是安全电子交易的早期支持者之一，最终采用了一种不同的协议，称为 3-D Secure，作为其客户安全数字支付和交易的框架。 3-D 安全方法是一种基于可扩展标记语言 (XML) 的协议，旨在成为在线信用卡和借记卡交易的附加安全层。

它最初由 Visa 和 Arcot Systems（现称为 CA Technologies）共同编写。 Mastercard、Discover 和 American Express 现在使用基于 3-D Secure 的类似协议。
