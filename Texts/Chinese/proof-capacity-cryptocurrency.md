# 容量证明（加密货币）
## 什么是加密货币的容量证明 (PoC)？

容量证明（PoC）是区块链中使用的一种[共识机制](/consensus-mechanism-cryptocurrency)算法，它允许网络中的[挖矿](/bitcoin-mining)设备使用其可用的硬盘空间来决定挖矿权并验证交易。这与使用挖矿设备的计算能力（如[工作量证明](/proof-work)算法）或矿工在加密货币中的股份（如权益[证明算法）形成对比](/proof-stake-pos)。

## 了解容量证明

容量证明成为解决工作量证明 (PoW) 系统中的高能耗问题和权益证明 (PoS) 系统中的加密货币囤积问题的众多替代解决方案之一。

[区块链网络](/blockchain)上的挖矿设备（也称为节点）使用其硬盘驱动器上的空白空间来挖掘可用的[加密货币](/cryptocurrency)。

PoC 不是像在 PoW 系统中那样反复更改块头中的数字和对解决方案值进行反复散列，而是通过在采矿活动开始之前将可能的解决方案列表存储在采矿设备的硬盘上来工作。

硬盘越大，硬盘上可以存储的可能解决方案值越多，矿工从他的列表中匹配所需哈希值的机会就越大，从而获得更多挖矿奖励的机会。

打个比方，如果彩票奖励是基于匹配中奖彩票上最多的数字，那么可能解决方案列表较长的玩家将有更好的获胜机会。此外，允许玩家一次又一次地继续使用彩票块号。

[Burstcoin](/burstcoin)是一种使用容量证明系统的加密货币。其他使用它的硬币是 Storj、Chia 和 SpaceMint。

## PoC 的工作原理：绘图和挖掘

容量证明协议涉及一个涉及绘图和挖掘的两步过程。

首先，绘制硬盘驱动器：所有可能的[nonce](/nonce)值列表是通过数据的重复散列创建的，包括矿工的帐户。每个这样的随机数包含 8192 个散列，编号从 0 到 8191。所有散列配对成“勺子”，这意味着相邻的散列组合成一对两个。例如，散列 0 和 1 构成勺子 0，散列 2 和 3 构成散列 1，依此类推。

第二步涉及实际的采矿练习，在此期间，矿工计算一个勺子数。例如，如果一个矿工开始采矿活动并生成一个 38 号勺子，那么该矿工将转到 nonce 1 的 38 号勺子，并使用该勺子的数据来计算截止日期值。

重复该过程以计算矿工硬盘上保存的每个随机数的截止日期。在计算完所有期限后，矿工选择具有最短期限的期限。

截止日期表示在允许矿工伪造新区块之前，自最后一个区块被伪造以来必须经过的时间（以秒为单位）。如果在这段时间内没有其他人伪造区块，矿工可以伪造区块并获得区块奖励。

例如，如果矿工 X 提出的最短期限为 36 秒，并且在接下来的 36 秒内没有其他矿工可以伪造该区块，则 X 将确保有机会伪造下一个区块并获得奖励。
